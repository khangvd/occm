    既存のストレージの管理 GitHubで編集 ドキュメントの変更をリクエストする

    02/27/2020 寄稿者

    Cloud Managerを使用すると、ボリューム、アグリゲート、CIFSサーバーを管理できます。また、容量の問題を回避するためにボリュームを移動するように求められます。

    既存のボリュームの管理

    ストレージのニーズの変化に応じて、既存のボリュームを管理できます。ボリュームを表示、編集、複製、復元、および削除できます。

    手順

    1. [Working Environments]ページで、ボリュームを管理するCloud Volumes ONTAP作業環境をダブルクリックします。

    2. ボリュームを管理します。

      仕事アクション

      ボリュームに関する情報を表示する

      ボリュームを選択し、[ 情報 ]をクリックします。

      ボリュームの編集（読み取り/書き込みボリュームのみ）

      1. ボリュームを選択し、[ 編集 ]をクリックします。

      2. ボリュームのスナップショットポリシー、NFSプロトコルバージョン、NFSアクセスコントロールリスト、または共有アクセス許可を変更し、[ 更新 ]をクリックします 。

      カスタムスナップショットポリシーが必要な場合は、System Managerを使用して作成できます。

      ボリュームを複製する

      1. ボリュームを選択し、[ クローン ]をクリックします。

      2. 必要に応じてクローン名を変更し、[ クローン ]をクリックします。

      このプロセスにより、FlexCloneボリュームが作成されます。 FlexCloneボリュームは書き込み可能なポイントインタイムコピーであり、メタデータに少量のスペースを使用し、データが変更または追加されたときにのみ追加スペースを消費するため、スペース効率が高くなります。

      FlexCloneボリュームの詳細については、 『 ONTAP 9 Logical Storage Management Guide』を参照してください。

      Snapshotコピーから新しいボリュームにデータを復元する

      1. ボリュームを選択し、[ Snapshotコピーから復元]をクリックします 。

      2. Snapshotコピーを選択し、新しいボリュームの名前を入力して、[ 復元 ]をクリックします。

      オンデマンドでSnapshotコピーを作成する

      1. ボリュームを選択し、[ Snapshotコピーの作成 ]をクリックします 。

      2. 必要に応じて名前を変更し、[ 作成 ]をクリックします 。

      NFSマウントコマンドを取得する

      1. ボリュームを選択し、[ マウントコマンド ]をクリックします。

      2. コピーをクリックします。

      基になるディスクタイプを変更する

      1. ボリュームを選択し、[ ディスクタイプと階層化ポリシーの変更 ]をクリックします。

      2. ディスクの種類を選択し、[ 変更 ]をクリックします。

      Cloud Managerは、選択したディスクタイプを使用する既存のアグリゲートにボリュームを移動するか、ボリュームの新しいアグリゲートを作成します。

      階層化ポリシーを変更する

      1. ボリュームを選択し、[ ディスクタイプと階層化ポリシーの変更 ]をクリックします。

      2. [ ポリシーの編集]をクリックします。

      3. 別のポリシーを選択して、「 変更」をクリックします。

      Cloud Managerは、選択したディスクタイプを階層化して使用する既存のアグリゲートにボリュームを移動するか、ボリュームの新しいアグリゲートを作成します。

      ボリュームのS3への同期を有効または無効にする

      ボリュームを選択し、[ S3に同期 ]または[同期関係を削除 ]をクリックします。

      これらのオプションを使用するには、S3機能への同期を有効にする必要があります。手順については、 AWS S3へのデータの同期を参照してください

      ボリュームを削除する

      1. ボリュームを選択し、[ 削除 ]をクリックします。

      2. もう一度[ 削除]をクリックして確認します。

    既存の集約の管理

    ディスクを追加し、アグリゲートに関する情報を表示し、それらを削除することにより、アグリゲートを自分で管理します。

    あなたが始める前に

    アグリゲートを削除する場合は、まずアグリゲート内のボリュームを削除しておく必要があります。

    このタスクについて

    アグリゲートのスペースが不足している場合、OnCommand System Managerを使用してボリュームを別のアグリゲートに移動できます。

    手順

    1. [Working Environments]ページで、アグリゲートを管理するCloud Volumes ONTAP作業環境をダブルクリックします。

    2. メニューアイコンをクリックし、[ 詳細設定 ] > [詳細割り当て ]をクリックします。

    3. 集計を管理します。

      仕事アクション

      集約に関する情報を表示する

      集計を選択して、[ 情報 ]をクリックします。

      特定のアグリゲートにボリュームを作成します

      アグリゲートを選択し、 ボリュームの作成をクリックします 。

      ディスクをアグリゲートに追加します

      1. アグリゲートを選択し、[ AWSディスクの追加 ]または[ Azureディスクの追加 ] をクリックします 。

      2. 追加し、[追加 ] をクリックしますディスクの数を選択します。

        アグリゲート内のすべてのディスクは同じサイズでなければなりません。

      集約を削除する

      1. ボリュームを含まないアグリゲートを選択し、[ 削除]をクリックします。

      2. もう一度[ 削除]をクリックして確認します。

    CIFSサーバーの変更

    DNSサーバーまたはActive Directoryドメインを変更する場合は、Cloud Volumes ONTAPでCIFSサーバーを変更して、引き続きクライアントにストレージを提供できるようにする必要があります。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[ 詳細 ] > [CIFSセットアップ ]をクリックします 。

    2. CIFSサーバーの設定を指定します。

      仕事アクション

      DNSプライマリおよびセカンダリIPアドレス

      CIFSサーバーの名前解決を提供するDNSサーバーのIPアドレス。

      リストされているDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバーを参加させるActive Directory（AD）ドメインのFQDN。

      ドメインへの参加を許可された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバーのNetBIOS名

      ADドメインで一意のCIFSサーバー名。

      組織単位

      CIFSサーバーに関連付けるADドメイン内の組織単位。デフォルトはCN = Computersです。
      AWS Managed Microsoft ADをCloud Volumes ONTAPのADサーバーとして設定する場合、このフィールドにOU = Computers、OU = corpと入力する必要があります 。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバー

      [ Active Directoryドメインを使用]を選択して、Active Directory DNSを使用してNTPサーバーを構成します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用する必要があります。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    3. 保存をクリックします 。

    結果

    Cloud Volumes ONTAPは、変更でCIFSサーバーを更新します。

    容量の問題を回避するためにボリュームを移動する

    Cloud Managerは、容量の問題を回避するためにボリュームを移動する必要があるが、問題を修正するための推奨事項を提供できないというアクションが必要なメッセージを表示する場合があります。この場合、問題を修正する方法を特定してから、1つ以上のボリュームを移動する必要があります。

    手順

    1. 問題の修正方法を特定します 。

    2. 分析に基づいて、容量の問題を回避するためにボリュームを移動します。

      • ボリュームを別のシステムに移動します 。

      • 同じシステム上の別のアグリゲートにボリュームを移動します 。

    容量の問題を修正する方法を特定する

    Cloud Managerが容量の問題を回避するためのボリュームの移動に関する推奨事項を提供できない場合、移動する必要があるボリュームと、同じシステム上の別のアグリゲートまたは別のシステムに移動する必要があるボリュームを識別する必要があります。

    手順

    1. 必要なアクションメッセージの詳細情報を表示して、容量制限に達した集約を特定します。

      たとえば、詳細情報には次のような情報が記載されている必要があります。Aggregate aggr1が容量制限に達しました。

    2. アグリゲートから移動する1つ以上のボリュームを特定します。

      1. 作業環境で、メニューアイコンをクリックし、[ 詳細 ] > [ 高度な割り当て ]をクリックします。

      2. 集約を選択し、[ 情報 ]をクリックします。

      3. ボリュームのリストを展開します。

      4. 各ボリュームのサイズを確認し、アグリゲートから移動する1つ以上のボリュームを選択します。

        将来的に追加の容量の問題を回避するために、アグリゲートのスペースを解放するのに十分な大きさのボリュームを選択する必要があります。

    3. システムがディスク制限に達していない場合は、同じシステム上の既存のアグリゲートまたは新しいアグリゲートにボリュームを移動する必要があります。

      詳細については、 容量の問題を回避するための別のアグリゲートへのボリュームの移動を参照してください。

    4. システムがディスク制限に達した場合、次のいずれかを実行します。

      1. 未使用のボリュームを削除します。

      2. アグリゲートの空きスペースにボリュームを再配置します。

        詳細については、 容量の問題を回避するための別のアグリゲートへのボリュームの移動を参照してください。

      3. 2つ以上のボリュームをスペースのある別のシステムに移動します。

        詳細については、 容量の問題を回避するための別のシステムへのボリュームの移動を参照してください。

    容量の問題を回避するためにボリュームを別のシステムに移動する

    1つ以上のボリュームを別のCloud Volumes ONTAPシステムに移動して、容量の問題を回避できます。システムがディスク制限に達した場合、これを行う必要があるかもしれません。

    このタスクについて

    このタスクの手順に従って、次のアクションが必要なメッセージを修正できます。 

    容量の問題を回避するには、ボリュームを移動する必要があります。ただし、システムがディスク制限に達したため、Cloud Managerはこのアクションを実行できません。
    手順

    1. 使用可能な容量があるCloud Volumes ONTAPシステムを特定するか、新しいシステムを展開します。

    2. ソース作業環境をターゲット作業環境にドラッグアンドドロップして、ボリュームの1回限りのデータレプリケーションを実行します。

      詳細については、 システム間でのデータの複製を参照してください。

    3. [レプリケーションステータス]ページに移動し、SnapMirror関係を解除して、レプリケートされたボリュームをデータ保護ボリュームから読み取り/書き込みボリュームに変換します。

      詳細については、 データ複製スケジュールと関係の管理を参照してください。

    4. データアクセス用のボリュームを構成します。

      データアクセス用のデスティネーションボリュームの設定については、 ONTAP 9 Volume Disaster Recovery Express Guideを参照してください 。

    5. 元のボリュームを削除します。

      詳しくは、 既存のボリュームの管理を参照してください。

    容量の問題を回避するためにボリュームを別のアグリゲートに移動する

    1つ以上のボリュームを別のアグリゲートに移動して、容量の問題を回避できます。

    このタスクについて

    このタスクの手順に従って、次のアクションが必要なメッセージを修正できます。 

    容量の問題を回避するには、2つ以上のボリュームを移動する必要があります。ただし、Cloud Managerはこのアクションを実行できません。
    手順

    1. 既存のアグリゲートに、移動する必要があるボリュームの空き容量があるかどうかを確認します。

      1. 作業環境で、メニューアイコンをクリックし、[ 詳細 ] > [ 高度な割り当て ]をクリックします。

      2. 各アグリゲートを選択し、[ 情報 ]をクリックしてから、使用可能な容量（アグリゲート容量から使用されたアグリゲート容量を引いたもの）を表示します。

    2. 必要に応じて、ディスクを既存のアグリゲートに追加します。

      1. アグリゲートを選択し、[ ディスクの追加 ] をクリックします 。

      2. 追加するディスクの数を選択し、[ 追加 ] をクリックします。

    3. 使用可能な容量があるアグリゲートがない場合は、新しいアグリゲートを作成します。

      詳細については、 集計の作成を参照してください。

    4. System ManagerまたはCLIを使用して、ボリュームをアグリゲートに移動します。

    5. ほとんどの場合、System Managerを使用してボリュームを移動できます。

      手順については、 ONTAP 9 Volume Move Express Guideを参照してください。
