    複数のAZのHAペア用のAWSトランジットゲートウェイのセットアップ GitHubで編集 ドキュメントの変更をリクエストする

    09/05/2019 寄稿者

    AWSトランジットゲートウェイを設定して、HAペアが存在するVPCの外部からHAペアのフローティングIPアドレスにアクセスできるようにします。

    Cloud Volumes ONTAP HA構成が複数のAWSアベイラビリティゾーンに分散している場合、VPC内からのNASデータアクセスにはフローティングIPアドレスが必要です。これらのフローティングIPアドレスは、障害が発生するとノード間で移行できますが、VPCの外部からネイティブにアクセスすることはできません。個別のプライベートIPアドレスは、VPCの外部からのデータアクセスを提供しますが、自動フェールオーバーは提供しません。

    クラスター管理インターフェイスとオプションのSVM管理LIFには、フローティングIPアドレスも必要です。

    AWSトランジットゲートウェイをセットアップする場合、HAペアが存在するVPCの外部からフローティングIPアドレスへのアクセスを有効にします。つまり、VPC外のNASクライアントとNetApp管理ツールは、フローティングIPにアクセスできます。

    中継ゲートウェイによって接続された2つのVPCを示す例を次に示します。 HAシステムは1つのVPCに存在し、クライアントは他のVPCに存在します。その後、フローティングIPアドレスを使用してクライアントにNASボリュームをマウントできます。

    次の手順は、同様の構成をセットアップする方法を示しています。

    手順

    1. トランジットゲートウェイを作成し、VPCをゲートウェイに接続します 。

    2. HAペアのフローティングIPアドレスを指定して、中継ゲートウェイのルートテーブルにルートを作成します。

      フローティングIPアドレスは、Cloud Managerの[作業環境情報]ページで確認できます。次に例を示します。

      次のサンプル画像は、中継ゲートウェイのルートテーブルを示しています。これには、Cloud Volumes ONTAPで使用される2つのVPCと4つのフローティングIPアドレスのCIDRブロックへのルートが含まれます。

    3. フローティングIPアドレスにアクセスする必要があるVPCのルートテーブルを変更します。

      1. フローティングIPアドレスにルートエントリを追加します。

      2. HAペアが存在するVPCのCIDRブロックにルートエントリを追加します。

        次のサンプル画像は、VPC 1へのルートとフローティングIPアドレスを含むVPC 2のルートテーブルを示しています。

    4. フローティングIPアドレスにアクセスする必要があるVPCにルートを追加して、HAペアのVPCのルートテーブルを変更します。

      VPC間のルーティングを完了するため、このステップは重要です。

      次のサンプル画像は、VPC 1のルートテーブルを示しています。これには、フローティングIPアドレスと、クライアントが存在するVPC 2へのルートが含まれています。 Cloud Managerは、HAペアを展開するときに、フローティングIPをルートテーブルに自動的に追加しました。

    5. フローティングIPアドレスを使用してクライアントにボリュームをマウントします。

      Cloud Managerで正しいIPアドレスを見つけるには、ボリュームを選択して[ コマンドのマウント ]をクリックします。
