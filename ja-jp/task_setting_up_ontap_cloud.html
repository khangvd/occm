Cloud Volumes ONTAPのセットアップ

2019年9月18日 寄稿者

Cloud Volumes ONTAPを展開した後、NTPを使用してシステム時刻を同期し、System ManagerまたはCLIからいくつかのオプションタスクを実行することにより、Cloud Volumes ONTAPをセットアップできます。

仕事説明

NTPを使用してシステム時刻を同期します

NTPサーバーを指定すると、ネットワーク内のシステム間の時刻が同期されます。これにより、時差による問題を防ぐことができます。

CIFSサーバーをセットアップするときに、Cloud Manager APIを使用するか、ユーザーインターフェースからNTPサーバーを指定します。

  • CIFSサーバーの変更

  • Cloud Manager API開発者ガイド

たとえば、AWSのシングルノードシステム用のAPIは次のとおりです。

オプション：AutoSupportの構成

AutoSupportはシステムの状態を予防的に監視し、デフォルトでNetAppテクニカルサポートにメッセージを自動的に送信します。

インスタンスを起動する前にアカウント管理者がプロキシサーバーをCloud Managerに追加した場合、Cloud Volumes ONTAPはAutoSupportメッセージにそのプロキシサーバーを使用するように構成されます。

AutoSupportをテストして、メッセージを送信できることを確認する必要があります。手順については、System ManagerヘルプまたはONTAP 9 System Administration Referenceを参照してください 。

オプション：EMSの構成

イベント管理システム（EMS）は、Cloud Volumes ONTAPシステムで発生するイベントに関する情報を収集して表示します。イベント通知を受信するには、特定のイベント重大度のイベント宛先（電子メールアドレス、SNMPトラップホスト、またはsyslogサーバー）およびイベントルートを設定できます。

CLIを使用してEMSを構成できます。手順については、 ONTAP 9 EMS Configuration Express Guideを参照してください。

オプション：複数のAWSアベイラビリティーゾーンにあるHAシステム用のSVM管理ネットワークインターフェース（LIF）を作成します

SnapCenterまたはSnapDrive for WindowsをHAペアで使用する場合は、ストレージ仮想マシン（SVM）管理ネットワークインターフェイス（LIF）が必要です。 SVM管理LIFは、複数のAWSアベイラビリティーゾーンでHAペアを使用する場合、 フローティング IPアドレスを使用する必要があります。

Cloud Managerは、HAペアを起動するときにフローティングIPアドレスを指定するように求めます。 IPアドレスを指定しなかった場合は、System ManagerまたはCLIから自分でSVM管理LIFを作成できます。次の例は、CLIからLIFを作成する方法を示しています。 

ネットワークインターフェイスcreate -vserver svm_cloud -lif svm_mgmt -role data -data-protocol none -home-node cloud-01 -home-port e0a -address 10.0.2.126 -netmask 255.255.255.0 -status-admin up -firewall-policy mgmt

オプション：構成ファイルのバックアップ場所を変更する

Cloud Volumes ONTAPは、適切に動作するために必要な構成可能なオプションに関する情報を含む構成バックアップファイルを自動的に作成します。

デフォルトでは、Cloud Volumes ONTAPは8時間ごとにファイルをCloud Managerホストにバックアップします。別の場所にバックアップを送信する場合は、データセンターまたはAWSでFTPまたはHTTPサーバーに場所を変更できます。たとえば、FASストレージシステムのバックアップ場所が既にある場合があります。

CLIを使用してバックアップの場所を変更できます。 ONTAP 9システム管理リファレンスを参照してください 。
