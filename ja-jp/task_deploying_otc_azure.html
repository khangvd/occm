    AzureでCloud Volumes ONTAPを起動する GitHubで編集 ドキュメントの変更をリクエストする

    02/03/2020 寄稿者

    Cloud ManagerでCloud Volumes ONTAP作業環境を作成することにより、AzureでシングルノードシステムまたはHAペアを起動できます。

    あなたが始める前に

    • 特に以前のリリースからアップグレードして初めてHAシステムを展開する場合は、Azureアカウントに必要なアクセス許可があることを確認してください。

      最新の権限は、 AzureのNetApp Cloud Centralポリシーにあります。

    • 構成を選択し、管理者からAzureネットワーク情報を取得する必要があります。詳細については、 Cloud Volumes ONTAP構成の計画を参照してください。

    • BYOLシステムを展開するには、各ノードに20桁のシリアル番号（ライセンスキー）が必要です。

    このタスクについて

    Cloud ManagerがAzureでCloud Volumes ONTAPシステムを作成すると、リソースグループ、ネットワークインターフェイス、ストレージアカウントなどのいくつかのAzureオブジェクトが作成されます。ウィザードの最後でリソースの概要を確認できます。

    手順

    1. [Working Environments]ページで、[ Create Cloud Volumes ONTAP ]をクリックして、プロンプトに従います。

    2. 作業環境の定義 ： Microsoft Azureを選択してから、単一のノードまたはHAペアを選択します。

    3. 詳細と資格情報 ：必要に応じて、Azureの資格情報とサブスクリプションを変更し、クラスター名とリソースグループ名を指定し、必要に応じてタグを追加してから、資格情報を指定します。

      次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      資格情報の編集

      それらをセットアップしてCloud Managerに追加した場合は、別のアカウントまたはサブスクリプションを選択できます。

      作業環境名

      Cloud Managerは、作業環境名を使用して、Cloud Volumes ONTAPシステムとAzure仮想マシンの両方に名前を付けます。また、そのオプションを選択した場合、定義済みのセキュリティグループのプレフィックスとして名前を使用します。

      リソースグループ名

      Use Defaultのチェックを外すと、新しいリソースグループの名前を入力できます。既存のリソースグループを使用する場合は、APIを使用する必要があります。

      タグ

      タグは、Azureリソースのメタデータです。 Cloud Managerは、タグをCloud Volumes ONTAPシステムおよびシステムに関連付けられた各Azureリソースに追加します。

      作業環境を作成するときに、ユーザーインターフェイスから最大4つのタグを追加できます。作成後にタグを追加できます。 APIは、作業環境を作成するときに4つのタグに制限しないことに注意してください。

      タグの詳細については、 Microsoft Azureドキュメント：タグを使用してAzureリソースを整理するを参照してください 。

      資格情報

      これらは、Cloud Volumes ONTAPクラスター管理者アカウントの資格情報です。これらの認証情報を使用して、OnCommand System ManagerまたはそのCLIを介してCloud Volumes ONTAPに接続できます。

    4. サービス ：このCloud Volumes ONTAPシステムで使用しない場合は、クラウドコンプライアンスを有効または無効にします。

      クラウドコンプライアンスの詳細をご覧ください 。

    5. ロケーションと接続性 ：ロケーションとセキュリティグループを選択し、チェックボックスを選択してCloud Managerとターゲットロケーション間のネットワーク接続を確認します。

    6. ライセンスおよびサポートサイトアカウント ：従量課金またはBYOLのどちらを使用するかを指定してから、NetAppサポートサイトアカウントを指定します。

      ライセンスの仕事、参照方法を理解するにはライセンス 。

      ネットアップサポートサイトアカウントは、従量課金制ではオプションですが、BYOLシステムでは必須です。 ネットアップサポートサイトアカウントを追加する方法を学びます 。

    7. 事前構成済みパッケージ ： パッケージの 1つを選択して、Cloud Volumes ONTAPシステムを迅速に展開するか、[ 独自の構成を作成 ]をクリックします 。

      パッケージのいずれかを選択した場合、ボリュームを指定して、構成を確認および承認するだけです。

    8. ライセンス ：必要に応じてCloud Volumes ONTAPバージョンを変更し、ライセンスを選択して、仮想マシンのタイプを選択します。

      システムの起動後にニーズが変わった場合は、後でライセンスまたは仮想マシンのタイプを変更できます。

      選択したバージョンで新しいリリース候補、一般提供、またはパッチリリースが利用可能な場合、Cloud Managerは、作業環境の作成時にシステムをそのバージョンに更新します。たとえば、Cloud Volumes ONTAP 9.6 RC1を選択し、9.6 GAが使用可能な場合、更新が発生します。 9.6から9.7など、リリースごとに更新は行われません。

    9. Azure Marketplaceからサブスクライブする ：Cloud ManagerがCloud Volumes ONTAPのプログラムによる展開を有効にできない場合は、手順に従ってください。

    10. 基礎となるストレージリソース ：初期集約の設定を選択します。ディスクタイプ、各ディスクのサイズ、およびBlobストレージへのデータ階層化を有効にするかどうかを選択します。

      次のことに注意してください。

      • ディスクタイプは初期ボリューム用です。後続のボリュームに別のディスクタイプを選択できます。

      • ディスクサイズは、初期アグリゲートのすべてのディスクと、シンプルプロビジョニングオプションを使用したときにCloud Managerが作成する追加アグリゲート用です。拡張割り当てオプションを使用して、異なるディスクサイズを使用するアグリゲートを作成できます。

        ディスクの種類とサイズの選択については、Azureでのシステムのサイズ設定をご覧ください。

      • ボリュームを作成または編集するときに、特定のボリューム階層化ポリシーを選択できます。

      • データ階層化を無効にすると、後続のアグリゲートで有効にできます。

        データ階層化の詳細をご覧ください 。

    11. ノーマルまたは高速書き込み速度を選択し、必要に応じて、多くの（WORM）ストレージを読んで追記をアクティブ： スピード＆WORM（シングルノードシステムのみ） 書きます 。

      書き込み速度の選択は、単一ノードシステムでのみサポートされます。

      書き込み速度の詳細をご覧ください 。

      WORMストレージの詳細をご覧ください 。

    12. ストレージとWORMへのセキュアな通信 （HAのみ）：必要に応じて、AzureストレージアカウントへのHTTPS接続を有効にし、1回書き込み、多数読み取り（WORM）ストレージをアクティブにするかどうかを選択します。

      HTTPS接続は、Cloud Volumes ONTAP 9.7 HAペアからAzureストレージアカウントへの接続です。このオプションを有効にすると、書き込みパフォーマンスに影響することに注意してください。作業環境を作成した後、設定を変更することはできません。

      WORMストレージの詳細をご覧ください 。

    13. ボリュームの作成 ：新しいボリュームの詳細を入力するか、[ スキップ ]をクリックします。

      iSCSIを使用する場合は、この手順をスキップする必要があります。 Cloud Managerでは、NFSおよびCIFSのみのボリュームを作成できます。

      このページのフィールドの一部は、一目瞭然です。次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にするかどうかに大きく依存します。これにより、現在使用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベルを制御できます（アクセス制御リストまたはACLとも呼ばれます）。ローカルまたはドメインのWindowsユーザーまたはグループ、またはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式を使用してユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されるNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、必要なストレージが最小限のポイントインタイムファイルシステムイメージです。デフォルトポリシーを選択することも、なしを選択することもできます。一時データにはnoneを選択できます。たとえば、Microsoft SQL Serverのtempdbです。

    14. CIFSセットアップ ：CIFSプロトコルを選択した場合は、CIFSサーバーをセットアップします。

      フィールド説明

      DNSプライマリおよびセカンダリIPアドレス

      CIFSサーバーの名前解決を提供するDNSサーバーのIPアドレス。
      リストされているDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバーを参加させるActive Directory（AD）ドメインのFQDN。

      ドメインへの参加を許可された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバーのNetBIOS名

      ADドメインで一意のCIFSサーバー名。

      組織単位

      CIFSサーバーに関連付けるADドメイン内の組織単位。デフォルトはCN = Computersです。

      Azure ADドメインサービスをCloud Volumes ONTAPのADサーバーとして構成するには、このフィールドにOU = AADDC ComputersまたはOU = AADDC Usersと入力する必要があります 。
      Azureドキュメント：Azure ADドメインサービスの管理対象ドメインに組織単位（OU）を作成する

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバー

      [ Active Directoryドメインを使用]を選択して、Active Directory DNSを使用してNTPサーバーを構成します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用する必要があります。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    15. 使用プロファイル、ディスクタイプ、および階層化ポリシー ：Storage Efficiency機能を有効にするかどうかを選択し、必要に応じてボリューム階層化ポリシーを変更します。

      詳細については、 「ボリューム使用プロファイルについて」および「 データ階層化の概要 」を参照してください。

    16. レビューと承認 ：選択内容をレビューして確認します。

      1. 構成に関する詳細を確認します。

      2. [ 詳細情報]をクリックして、Cloud Managerが購入するサポートとAzureリソースの詳細を確認します。

      3. 私を理解し...チェックボックスを選択します。

      4. Goをクリックします。

    結果

    Cloud ManagerはCloud Volumes ONTAPシステムを展開します。タイムラインで進行状況を追跡できます。

    Cloud Volumes ONTAPシステムの展開で問題が発生した場合は、エラーメッセージを確認してください。作業環境を選択して、[環境の再作成 ]をクリックすることもできます 。

    さらにヘルプが必要な場合は、 NetApp Cloud Volumes ONTAP Supportにアクセスしてください。

    終わったら

    • CIFS共有をプロビジョニングした場合、ユーザーまたはグループにファイルとフォルダーへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    • ボリュームにクォータを適用する場合は、System ManagerまたはCLIを使用します。

      クォータを使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。
