    Cloud Centralアカウント GitHubで編集 ドキュメントの変更をリクエストする

    10/31/2019 寄稿者

    各Cloud Managerシステムは、 NetApp Cloud Centralアカウントに関連付けられています 。 Cloud Centralアカウントはマルチテナンシーを提供し、分離されたワークスペースでユーザーとリソースを整理できます。

    Cloud Centralアカウントはマルチテナンシーを可能にします：

    • 1つのCloud Centralアカウントには、さまざまなビジネスニーズに対応する複数のCloud Managerシステムを含めることができます。

      ユーザーはCloud Centralアカウントに関連付けられているため、個々のCloud Managerシステムごとにユーザーを構成する必要はありません。

    • 各Cloud Managerシステム内で、複数のユーザーがワークスペースと呼ばれる分離された環境でCloud Volumes ONTAPシステムを展開および管理できます。

      これらのワークスペースは、共有されない限り、他のユーザーには見えません。

    Cloud Managerをデプロイするときに、Cloud Centralアカウントを選択してシステムに関連付けます。

    アカウント管理者は、ユーザー、ワークスペース、およびサービスコネクタを管理することにより、このアカウントの設定を変更できます。

    段階的な手順については、Cloud Centralアカウントのセットアップを参照してください。

    Cloud Centralアカウントサービスに接続するには、Cloud Managerがhttps://cloudmanager.cloud.netapp.comにアクセスする必要があります。ファイアウォールでこのURLを開き、Cloud Managerがサービスに接続できることを確認します。

    ユーザー、ワークスペース、およびサービスコネクタ

    Cloud Managerのアカウント設定ウィジェットを使用すると、アカウント管理者はCloud Centralアカウントを管理できます。アカウントを作成したばかりの場合は、最初から始めます。ただし、既にアカウントを設定している場合は、そのアカウントに関連付けられているすべてのユーザー、ワークスペース、およびサービスコネクタが表示されます。

    ユーザー

    これらは、Cloud Centralアカウントに関連付けるNetApp Cloud Centralユーザーです。ユーザーとアカウントおよびそのアカウントの1つ以上のワークスペースを関連付けると、それらのユーザーはCloud Managerで作業環境を作成および管理できます。

    ユーザーを関連付けるとき、ユーザーにロールを割り当てます。

    • アカウント管理者 ：Cloud Managerで任意のアクションを実行できます。

    • ワークスペース管理者 ：割り当てられたワークスペースでリソースを作成および管理できます。

    ワークスペース

    Cloud Managerでは、ワークスペースは、任意の数の作業環境を他の作業環境から分離します。アカウント管理者がワークスペースに管理者を関連付けない限り、ワークスペース管理者はワークスペースの作業環境にアクセスできません。

    作業環境はストレージシステムを表します。

    • シングルノードCloud Volumes ONTAPシステムまたはHAペア

    • ネットワーク内のオンプレミスONTAPクラスター

    • NetApp Private Storage構成のONTAPクラスター

    サービスコネクタ

    サービスコネクタはCloud Managerの一部です。接続するいくつかのCloud Centralサービス（auth0およびCloud Centralアカウント）を除き、Cloud Managerソフトウェアの大部分（ユーザーインターフェイスなど）を実行します。サービスコネクタは、クラウドプロバイダーに展開された仮想マシンインスタンス、または構成したオンプレミスホストで実行されます。

    サービスコネクタを複数のNetAppクラウドデータサービスで使用できます。たとえば、Cloud Managerのサービスコネクタが既にある場合は、Cloud Tieringサービスをセットアップするときに選択できます。

    例

    次の例は、2つのワークスペースを使用してCloud Volumes ONTAPシステムの分離環境を作成するアカウントを示しています。たとえば、1つのワークスペースはステージング環境用で、もう1つのワークスペースは実稼働環境用です。

    Cloud ManagerおよびCloud Volumes ONTAPシステムは、実際にはNetApp Cloud Centralアカウントに存在するのではなく、クラウドプロバイダーで実行されています。これは、各コンポーネント間の関係の概念的な表現です。

    2つの個別のCloud Centralアカウントを使用して、最高レベルのマルチテナンシーを示す別の例を次に示します。たとえば、サービスプロバイダーは1つのCloud CentralアカウントでCloud Managerを使用して顧客にサービスを提供し、別のアカウントを使用してビジネスユニットの1つに災害復旧を提供できます。

    アカウント2には2つの個別のサービスコネクタが含まれていることに注意してください。これは、別々の地域または別々のクラウドプロバイダーにシステムがある場合に発生する可能性があります。

    繰り返しますが、Cloud ManagerとCloud Volumes ONTAPシステムは、実際にはNetApp Cloud Centralアカウントに存在するのではなく、クラウドプロバイダーで実行されています。これは、各コンポーネント間の関係の概念的な表現です。

    Cloud Centralアカウントとの統合に関するFAQ

    Cloud Manager 3.7にアップグレードした後、NetAppは特定のCloud Managerシステムを選択してCloud Centralアカウントと統合します。このFAQは、プロセスについての疑問に答えることができます。

    プロセスにはどれくらい時間がかかりますか？

    ほんの数分。

    Cloud Managerは使用できなくなりますか？

    いいえ、引き続きCloud Managerシステムにアクセスできます。

    Cloud Volumes ONTAPはどうですか？

    Cloud Volumes ONTAPシステムに混乱はありません。

    このプロセス中に何が起こりますか？

    NetAppは、統合プロセス中に次のことを行います。

    1. 新しいCloud Centralアカウントを作成し、Cloud Managerシステムに関連付けます。

    2. 既存の各ユーザーに新しい役割を割り当てます。

      • Cloud Manager管理者はアカウント管理者になります

      • テナント管理者と作業環境管理者がワークスペース管理者になります

    3. 既存のテナントを置き換えるワークスペースを作成します。

    4. これらのワークスペースに作業環境を配置します。

    5. サービスコネクタをすべてのワークスペースに関連付けます。

    Cloud Managerシステムをインストールした場所は重要ですか？

    いいえ。NetAppは、AWS、Azure、またはオンプレミスのどこにいるかに関係なく、システムがCloud Centralアカウントと統合されます。
