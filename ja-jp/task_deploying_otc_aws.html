    AWSでCloud Volumes ONTAPを起動する GitHubで編集 ドキュメントの変更をリクエストする

    02/03/2020 寄稿者

    Cloud Volumes ONTAPは、単一システム構成で、またはAWSでHAペアとして起動できます。

    AWS Marketplaceからサブスクライブする

    AWS Marketplaceからサブスクライブして、Cloud Volumes ONTAPの料金を支払うか、Cloud Volumes ONTAP BYOLをデプロイできます。

    PAYGOの購読

    Cloud Volumes ONTAPの無料トライアルが終了した後、サービスが中断しないように、AWS Marketplaceからサブスクライブします。作成したすべてのCloud Volumes ONTAP 9.6以降のPAYGOシステムと、有効にした各アドオン機能に対して、このサブスクリプションから課金されます。

    次のビデオは、サブスクリプションプロセスを示しています。

    複数のIAMユーザーが同じAWSアカウントで作業している場合、各ユーザーがサブスクライブする必要があります。最初のユーザーがサブスクライブすると、次の画像に示すように、AWSは後続のユーザーに既にサブスクライブしていることを示します。 AWSアカウントのサブスクリプションが設定されている間、各IAMユーザーは自分自身をサブスクリプションに関連付ける必要があります。以下に示すメッセージが表示された場合は、 ここをクリックしてリンクをクリックしてCloud Centralに移動し、プロセスを完了してください。

    BYOLのサブスクライブ

    独自のライセンス（BYOL）を持ってCloud Volumes ONTAPを起動する場合は、AWS Marketplaceでその製品をサブスクライブする必要があります 。

    各AWS Marketplaceページの詳細をご覧ください 。

    AWSでシングルノードCloud Volumes ONTAPシステムを起動する

    AWSでCloud Volumes ONTAPを起動するには、Cloud Managerで新しい作業環境を作成する必要があります。

    あなたが始める前に

    • 設定を選択し、管理者からAWSネットワーク情報を取得して準備しておく必要があります。詳細については、 Cloud Volumes ONTAP構成の計画を参照してください。

    • BYOLシステムを起動するには、20桁のシリアル番号（ライセンスキー）が必要です。

    • CIFSを使用する場合は、DNSとActive Directoryをセットアップしておく必要があります。詳細については、 AWSのCloud Volumes ONTAPのネットワーク要件を参照してください。

    このタスクについて

    作業環境を作成した直後に、Cloud Managerは指定されたVPCでテストインスタンスを起動して接続を確認します。成功すると、Cloud Managerはすぐにインスタンスを終了し、Cloud Volumes ONTAPシステムのデプロイを開始します。 Cloud Managerが接続を確認できない場合、作業環境の作成は失敗します。テストインスタンスは、t2.nano（デフォルトのVPCテナント用）またはm3.medium（専用VPCテナント用）のいずれかです。

    手順

    1. [Working Environments]ページで、[ Create Cloud Volumes ONTAP ]をクリックして、プロンプトに従います。

    2. 作業環境の定義 ： Amazon Web ServicesおよびCloud Volumes ONTAPを選択します 。

    3. 詳細と認証情報 ：オプションで、AWS認証情報とサブスクリプションを変更し、作業環境名を入力し、必要に応じてタグを追加してから、パスワードを入力します。

      このページのフィールドの一部は、一目瞭然です。次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      資格情報の編集

      このCloud Volumes ONTAPシステムで使用するAWS認証情報とマーケットプレイスサブスクリプションを選択します。

      従量制のCloud Volumes ONTAPシステムを作成するには、AWS MarketplaceからCloud Volumes ONTAPへのサブスクリプションに関連付けられているAWS認証情報を選択する必要があります。

      [ サブスクリプションの追加]をクリックして、選択した資格情報をサブスクリプションに関連付けます。

      追加のAWS認証情報をCloud Managerに追加する方法を学びます 。

      作業環境名

      Cloud Managerは、作業環境名を使用して、Cloud Volumes ONTAPシステムとAmazon EC2インスタンスの両方に名前を付けます。また、そのオプションを選択した場合、定義済みのセキュリティグループのプレフィックスとして名前を使用します。

      タグを追加する

      AWSタグは、AWSリソースのメタデータです。 Cloud Managerは、タグをCloud Volumes ONTAPインスタンスと、インスタンスに関連付けられた各AWSリソースに追加します。

      作業環境を作成するときに、ユーザーインターフェイスから最大4つのタグを追加できます。作成後にタグを追加できます。 APIは、作業環境を作成するときに4つのタグに制限しないことに注意してください。

      タグの詳細については、 AWSドキュメント：Amazon EC2リソースのタグ付けを参照してください 。

      資格情報

      これらは、Cloud Volumes ONTAPクラスター管理者アカウントの資格情報です。これらの認証情報を使用して、OnCommand System ManagerまたはそのCLIを介してCloud Volumes ONTAPに接続できます。

    4. サービス ：このCloud Volumes ONTAPシステムで使用したくない個々のサービスを有効または無効にします。

      • クラウドコンプライアンスの詳細をご覧ください 。

      • S3へのバックアップの詳細をご覧ください 。

    5. 場所と接続性 ：AWSワークシートに記録したネットワーク情報を入力します。

      次の画像は、記入されたページを示しています。

    6. データ暗号化 ：データ暗号化なしまたはAWS管理の暗号化を選択します。

      AWS管理の暗号化では、アカウントまたは別のAWSアカウントから異なるカスタマーマスターキー（CMK）を選択できます。

      Cloud Volumes ONTAPシステムを作成した後、AWSデータ暗号化方式を変更することはできません。

      Cloud Volumes ONTAP用にAWS KMSをセットアップする方法を学びます 。

      サポートされている暗号化技術の詳細をご覧ください 。

    7. ライセンスおよびサポートサイトアカウント ：従量課金またはBYOLのどちらを使用するかを指定してから、NetAppサポートサイトアカウントを指定します。

      ライセンスの仕事、参照方法を理解するにはライセンス 。

      ネットアップサポートサイトアカウントは、従量課金制ではオプションですが、BYOLシステムでは必須です。 ネットアップサポートサイトアカウントを追加する方法を学びます 。

    8. 事前構成済みパッケージ ：いずれかのパッケージを選択してCloud Volumes ONTAPをすばやく起動するか、[ 独自の構成を作成 ]をクリックします 。

      パッケージのいずれかを選択した場合、ボリュームを指定して、構成を確認および承認するだけです。

    9. IAMロール ：Cloud Managerにロールを作成させるデフォルトのオプションを維持する必要があります。

      独自のポリシーを使用する場合は、Cloud Volumes ONTAPノードのポリシー要件を満たす必要があります 。

    10. ライセンス ：必要に応じてCloud Volumes ONTAPバージョンを変更し、ライセンス、インスタンスタイプ、およびインスタンステナントを選択します。

      インスタンスの起動後にニーズが変わった場合は、後でライセンスまたはインスタンスタイプを変更できます。

      選択したバージョンで新しいリリース候補、一般提供、またはパッチリリースが利用可能な場合、Cloud Managerは、作業環境の作成時にシステムをそのバージョンに更新します。たとえば、Cloud Volumes ONTAP 9.6 RC1を選択し、9.6 GAが使用可能な場合、更新が発生します。 9.6から9.7など、リリースごとに更新は行われません。

    11. 基礎となるストレージリソース ：初期アグリゲートの設定を選択します：ディスクタイプ、各ディスクのサイズ、データ階層化を有効にするかどうか。

      次のことに注意してください。

      • ディスクタイプは初期ボリューム用です。後続のボリュームに別のディスクタイプを選択できます。

      • ディスクサイズは、初期アグリゲートのすべてのディスクと、シンプルプロビジョニングオプションを使用したときにCloud Managerが作成する追加アグリゲート用です。拡張割り当てオプションを使用して、異なるディスクサイズを使用するアグリゲートを作成できます。

        ディスクのタイプとサイズの選択については、AWSでのシステムのサイズ設定をご覧ください。

      • ボリュームを作成または編集するときに、特定のボリューム階層化ポリシーを選択できます。

      • データ階層化を無効にすると、後続のアグリゲートで有効にできます。

        データ階層化の仕組みを学びます 。

    12. スピード＆WORMを書く ：必要に応じて、 ノーマルまたは高速書き込み速度を選択し、アクティブ書き込み後、多くの（WORM）ストレージをお読みください。

      書き込み速度の詳細をご覧ください 。

      WORMストレージの詳細をご覧ください 。

    13. ボリュームの作成 ：新しいボリュームの詳細を入力するか、[ スキップ ]をクリックします。

      iSCSI用のボリュームを作成する場合は、この手順をスキップできます。 Cloud Managerは、NFSおよびCIFSのみのボリュームをセットアップします。

      このページのフィールドの一部は、一目瞭然です。次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にするかどうかに大きく依存します。これにより、現在使用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベルを制御できます（アクセス制御リストまたはACLとも呼ばれます）。ローカルまたはドメインのWindowsユーザーまたはグループ、またはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式を使用してユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されるNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、必要なストレージが最小限のポイントインタイムファイルシステムイメージです。デフォルトポリシーを選択することも、なしを選択することもできます。一時データにはnoneを選択できます。たとえば、Microsoft SQL Serverのtempdbです。

      次の画像は、CIFSプロトコル用に記入されたボリュームページを示しています。

    14. CIFSセットアップ ：CIFSプロトコルを選択した場合は、CIFSサーバーをセットアップします。

      フィールド説明

      DNSプライマリおよびセカンダリIPアドレス

      CIFSサーバーの名前解決を提供するDNSサーバーのIPアドレス。
      リストされているDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバーを参加させるActive Directory（AD）ドメインのFQDN。

      ドメインへの参加を許可された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバーのNetBIOS名

      ADドメインで一意のCIFSサーバー名。

      組織単位

      CIFSサーバーに関連付けるADドメイン内の組織単位。デフォルトはCN = Computersです。
      AWS Managed Microsoft ADをCloud Volumes ONTAPのADサーバーとして設定する場合、このフィールドにOU = Computers、OU = corpと入力する必要があります 。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバー

      [ Active Directoryドメインを使用]を選択して、Active Directory DNSを使用してNTPサーバーを構成します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用する必要があります。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    15. 使用プロファイル、ディスクタイプ、および階層化ポリシー ：Storage Efficiency機能を有効にするかどうかを選択し、必要に応じてボリューム階層化ポリシーを編集します。

      詳細については、 「ボリューム使用プロファイルについて」および「 データ階層化の概要 」を参照してください。

    16. レビューと承認 ：選択内容をレビューして確認します。

      1. 構成に関する詳細を確認します。

      2. [ 詳細]をクリックして、Cloud Managerが購入するサポートとAWSリソースの詳細を確認します。

      3. 私を理解し...チェックボックスを選択します。

      4. Goをクリックします。

    結果

    Cloud ManagerはCloud Volumes ONTAPインスタンスを起動します。タイムラインで進行状況を追跡できます。

    Cloud Volumes ONTAPインスタンスの起動に問題が発生した場合は、エラーメッセージを確認してください。作業環境を選択して、[環境を再作成]をクリックすることもできます。

    さらにヘルプが必要な場合は、 NetApp Cloud Volumes ONTAP Supportにアクセスしてください。

    終わったら

    • CIFS共有をプロビジョニングした場合、ユーザーまたはグループにファイルとフォルダーへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    • ボリュームにクォータを適用する場合は、System ManagerまたはCLIを使用します。

      クォータを使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。

    AWSでCloud Volumes ONTAP HAペアを起動する

    AWSでCloud Volumes ONTAP HAペアを起動する場合は、Cloud ManagerでHA作業環境を作成する必要があります。

    あなたが始める前に

    • 設定を選択し、管理者からAWSネットワーク情報を取得して準備しておく必要があります。詳細については、 Cloud Volumes ONTAP構成の計画を参照してください。

    • BYOLライセンスを購入した場合、各ノードに20桁のシリアル番号（ライセンスキー）が必要です。

    • CIFSを使用する場合は、DNSとActive Directoryをセットアップしておく必要があります。詳細については、 AWSのCloud Volumes ONTAPのネットワーク要件を参照してください。

    このタスクについて

    作業環境を作成した直後に、Cloud Managerは指定されたVPCでテストインスタンスを起動して接続を確認します。成功すると、Cloud Managerはすぐにインスタンスを終了し、Cloud Volumes ONTAPシステムのデプロイを開始します。 Cloud Managerが接続を確認できない場合、作業環境の作成は失敗します。テストインスタンスは、t2.nano（デフォルトのVPCテナント用）またはm3.medium（専用VPCテナント用）のいずれかです。

    手順

    1. [Working Environments]ページで、[ Create Cloud Volumes ONTAP ]をクリックして、プロンプトに従います。

    2. 作業環境の定義 ： Amazon Web ServicesおよびCloud Volumes ONTAP HAを選択します 。

    3. 詳細と認証情報 ：オプションで、AWS認証情報とサブスクリプションを変更し、作業環境名を入力し、必要に応じてタグを追加してから、パスワードを入力します。

      このページのフィールドの一部は、一目瞭然です。次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      資格情報の編集

      このCloud Volumes ONTAPシステムで使用するAWS認証情報とマーケットプレイスサブスクリプションを選択します。

      従量制のCloud Volumes ONTAPシステムを作成するには、AWS MarketplaceからCloud Volumes ONTAPへのサブスクリプションに関連付けられているAWS認証情報を選択する必要があります。

      [ サブスクリプションの追加]をクリックして、選択した資格情報をサブスクリプションに関連付けます。

      追加のAWS認証情報をCloud Managerに追加する方法を学びます 。

      作業環境名

      Cloud Managerは、作業環境名を使用して、Cloud Volumes ONTAPシステムとAmazon EC2インスタンスの両方に名前を付けます。また、そのオプションを選択した場合、定義済みのセキュリティグループのプレフィックスとして名前を使用します。

      タグを追加する

      AWSタグは、AWSリソースのメタデータです。 Cloud Managerは、タグをCloud Volumes ONTAPインスタンスと、インスタンスに関連付けられた各AWSリソースに追加します。

      作業環境を作成するときに、ユーザーインターフェイスから最大4つのタグを追加できます。作成後にタグを追加できます。 APIは、作業環境を作成するときに4つのタグに制限しないことに注意してください。

      タグの詳細については、 AWSドキュメント：Amazon EC2リソースのタグ付けを参照してください 。

      資格情報

      これらは、Cloud Volumes ONTAPクラスター管理者アカウントの資格情報です。これらの認証情報を使用して、OnCommand System ManagerまたはそのCLIを介してCloud Volumes ONTAPに接続できます。

    4. サービス ：このCloud Volumes ONTAPシステムで使用したくない個々のサービスを有効または無効にします。

      • S3へのバックアップの詳細をご覧ください 。

      • クラウドコンプライアンスの詳細をご覧ください 。

    5. HA展開モデル ：HA構成を選択します。

      展開モデルの概要については、 Cloud Volumes ONTAP HA for AWSを参照してください。

    6. リージョンとVPC ：AWSワークシートに記録したネットワーク情報を入力します。

      次の図は、複数のAZ構成用に記入されたページを示しています。

    7. 接続性とSSH認証 ：HAペアとメディエーターの接続方法を選択します。

    8. フローティングIP ：複数のAZを選択した場合、フローティングIPアドレスを指定します。

      IPアドレスは、リージョン内のすべてのVPCのCIDRブロックの外側にある必要があります。詳細については、 複数のAZでのCloud Volumes ONTAP HAのAWSネットワーキング要件を参照してください。

    9. ルートテーブル ：複数のAZを選択した場合、フローティングIPアドレスへのルートを含める必要があるルートテーブルを選択します。

      ルートテーブルが複数ある場合は、正しいルートテーブルを選択することが非常に重要です。そうしないと、一部のクライアントがCloud Volumes ONTAP HAペアにアクセスできない場合があります。ルートテーブルの詳細については、 AWSドキュメント：Route Tablesを参照してください。

    10. データ暗号化 ：データ暗号化なしまたはAWS管理の暗号化を選択します。

      AWS管理の暗号化では、アカウントまたは別のAWSアカウントから異なるカスタマーマスターキー（CMK）を選択できます。

      Cloud Volumes ONTAPシステムを作成した後、AWSデータ暗号化方式を変更することはできません。

      Cloud Volumes ONTAP用にAWS KMSをセットアップする方法を学びます 。

      サポートされている暗号化技術の詳細をご覧ください 。

    11. ライセンスおよびサポートサイトアカウント ：従量課金またはBYOLのどちらを使用するかを指定してから、NetAppサポートサイトアカウントを指定します。

      ライセンスの仕事、参照方法を理解するにはライセンス 。

      ネットアップサポートサイトアカウントは、従量課金制ではオプションですが、BYOLシステムでは必須です。 ネットアップサポートサイトアカウントを追加する方法を学びます 。

    12. 事前構成済みパッケージ ：いずれかのパッケージを選択してCloud Volumes ONTAPシステムをすばやく起動するか、[ 独自の構成を作成 ]をクリックします 。

      パッケージのいずれかを選択した場合、ボリュームを指定して、構成を確認および承認するだけです。

    13. IAMロール ：デフォルトオプションを保持して、Cloud Managerにロールを作成させます。

      独自のポリシーを使用する場合は、Cloud Volumes ONTAPノードとHAメディエーターのポリシー要件を満たす必要があります 。

    14. ライセンス ：必要に応じてCloud Volumes ONTAPバージョンを変更し、ライセンス、インスタンスタイプ、およびインスタンステナントを選択します。

      インスタンスの起動後にニーズが変わった場合は、後でライセンスまたはインスタンスタイプを変更できます。

      選択したバージョンで新しいリリース候補、一般提供、またはパッチリリースが利用可能な場合、Cloud Managerは、作業環境の作成時にシステムをそのバージョンに更新します。たとえば、Cloud Volumes ONTAP 9.6 RC1を選択し、9.6 GAが使用可能な場合、更新が発生します。 9.6から9.7など、リリースごとに更新は行われません。

    15. 基礎となるストレージリソース ：初期アグリゲートの設定を選択します：ディスクタイプ、各ディスクのサイズ、データ階層化を有効にするかどうか。

      次のことに注意してください。

      • ディスクタイプは初期ボリューム用です。後続のボリュームに別のディスクタイプを選択できます。

      • ディスクサイズは、初期アグリゲートのすべてのディスクと、シンプルプロビジョニングオプションを使用したときにCloud Managerが作成する追加アグリゲート用です。拡張割り当てオプションを使用して、異なるディスクサイズを使用するアグリゲートを作成できます。

        ディスクのタイプとサイズの選択については、AWSでのシステムのサイズ設定をご覧ください。

      • ボリュームを作成または編集するときに、特定のボリューム階層化ポリシーを選択できます。

      • データ階層化を無効にすると、後続のアグリゲートで有効にできます。

        データ階層化の仕組みを学びます 。

    16. WORM ：必要に応じて、1回書き込み、多数読み取り（WORM）ストレージをアクティブにします。

      WORMストレージの詳細をご覧ください 。

    17. ボリュームの作成 ：新しいボリュームの詳細を入力するか、[ スキップ ]をクリックします。

      iSCSI用のボリュームを作成する場合は、この手順をスキップできます。 Cloud Managerは、NFSおよびCIFSのみのボリュームをセットアップします。

      このページのフィールドの一部は、一目瞭然です。次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にするかどうかに大きく依存します。これにより、現在使用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベルを制御できます（アクセス制御リストまたはACLとも呼ばれます）。ローカルまたはドメインのWindowsユーザーまたはグループ、またはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式を使用してユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されるNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、必要なストレージが最小限のポイントインタイムファイルシステムイメージです。デフォルトポリシーを選択することも、なしを選択することもできます。一時データにはnoneを選択できます。たとえば、Microsoft SQL Serverのtempdbです。

      次の画像は、CIFSプロトコル用に記入されたボリュームページを示しています。

    18. CIFSセットアップ ：CIFSプロトコルを選択した場合は、CIFSサーバーをセットアップします。

      フィールド説明

      DNSプライマリおよびセカンダリIPアドレス

      CIFSサーバーの名前解決を提供するDNSサーバーのIPアドレス。
      リストされているDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバーを参加させるActive Directory（AD）ドメインのFQDN。

      ドメインへの参加を許可された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するのに十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバーのNetBIOS名

      ADドメインで一意のCIFSサーバー名。

      組織単位

      CIFSサーバーに関連付けるADドメイン内の組織単位。デフォルトはCN = Computersです。
      AWS Managed Microsoft ADをCloud Volumes ONTAPのADサーバーとして設定する場合、このフィールドにOU = Computers、OU = corpと入力する必要があります 。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバー

      [ Active Directoryドメインを使用]を選択して、Active Directory DNSを使用してNTPサーバーを構成します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用する必要があります。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    19. 使用プロファイル、ディスクタイプ、および階層化ポリシー ：Storage Efficiency機能を有効にするかどうかを選択し、必要に応じてボリューム階層化ポリシーを編集します。

      詳細については、 「ボリューム使用プロファイルについて」および「 データ階層化の概要 」を参照してください。

    20. レビューと承認 ：選択内容をレビューして確認します。

      1. 構成に関する詳細を確認します。

      2. [ 詳細]をクリックして、Cloud Managerが購入するサポートとAWSリソースの詳細を確認します。

      3. 私を理解し...チェックボックスを選択します。

      4. Goをクリックします。

    結果

    Cloud ManagerはCloud Volumes ONTAP HAペアを起動します。タイムラインで進行状況を追跡できます。

    HAペアの起動で問題が発生した場合は、失敗メッセージを確認してください。作業環境を選択して、[環境を再作成]をクリックすることもできます。

    さらにヘルプが必要な場合は、 NetApp Cloud Volumes ONTAP Supportにアクセスしてください。

    終わったら

    • CIFS共有をプロビジョニングした場合、ユーザーまたはグループにファイルとフォルダーへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    • ボリュームにクォータを適用する場合は、System ManagerまたはCLIを使用します。

      クォータを使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。
