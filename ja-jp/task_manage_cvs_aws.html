    AWS向けCloud Volumesサービスの管理 GitHubで編集 ドキュメントの変更をリクエストする

    09/03/2019 寄稿者

    Cloud Managerを使用すると、AWSサブスクリプションのCloud Volumes ServiceでNFSクラウドボリュームを検出できます。検出後、追加のNFSクラウドボリュームをCloud Managerから直接追加できます。

    Cloud Managerは、Cloud Volumes Service for AWSでSMBまたはデュアルプロトコルボリュームをサポートしていません。

    始める前に

    • Cloud Managerを使用すると、AWSサブスクリプションの既存の Cloud Volumesサービスを検出できます。サブスクリプションをまだセットアップしていない場合は、 NetApp Cloud Volumes Service for AWSアカウントセットアップガイドを参照してください。

      Cloud Managerでリージョンを検出する前に、リージョンごとにこのセットアッププロセスに従い、Cloud Volumes Serviceから最初のボリュームをプロビジョニングする必要があります。

    • Cloud Managerに提供できるように、Cloud Volumes APIキーとシークレットキーを取得する必要があります。 手順については、Cloud Volumes Service for AWSのドキュメントを参照してください 。

    AWSサブスクリプション用のCloud Volumesサービスの発見

    開始するには、AWSリージョンでクラウドボリュームを検出する必要があります。後で追加の地域を発見できます。

    手順

    1. [作業環境]ページで、[ 検出 ]をクリックします。

    2. AWSのCloud Volumes Serviceを選択します。

    3. Cloud Volumes Serviceサブスクリプションに関する情報を提供します。

      1. クラウドボリュームが存在するAWSリージョンを選択します。

      2. Cloud Volumes APIキーとシークレットキーを入力します。 手順については、Cloud Volumes Service for AWSのドキュメントを参照してください 。

      3. Goをクリックします。

    結果

    これで、Cloud Managerは、Working EnvironmentsページにAWS構成のCloud Volumesサービスを表示するはずです。

    追加の地域を発見する

    追加のリージョンにクラウドボリュームがある場合、個々のリージョンを検出する必要があります。

    手順

    1. [作業環境]ページで、作業環境を選択します（ただし、ダブルクリックして開かないでください）。

    2. 右側のペインで、[ 別の地域のCloud Volumesサービスの検出 ]をクリックします。

    3. Cloud Volumes Serviceサブスクリプションに関する情報を提供します。

      1. クラウドボリュームが存在するAWSリージョンを選択します。

      2. Cloud Volumes APIキーとシークレットキーを入力します。 手順については、Cloud Volumes Service for AWSのドキュメントを参照してください 。

      3. Goをクリックします。

    結果

    Cloud Managerは、選択したリージョンのクラウドボリュームに関する情報を検出します。

    クラウドボリュームの作成

    Cloud Managerを使用すると、NFSv3クラウドボリュームを作成できます。クラウドボリュームは、初期サイズと単一のエクスポートポリシーでのみ作成できます。ボリュームの編集は、Cloud Volume Serviceユーザーインターフェイスから行う必要があります。

    1. 作業環境を開きます。

    2. [ 新しいボリュームの追加]をクリックします。

    3. ボリュームに関する詳細を入力します。

      1. ボリュームの名前を入力します。

      2. 100 GiB〜90,000 GiB（88 TiBに相当）の範囲内のサイズを指定します。

        Cloud ManagerはボリュームをGiBで表示し、Cloud VolumesサービスはボリュームをGBで表示します。

      3. サービスレベルを指定します：Standard、Premium、またはExtreme。

        これらのサービスレベルの詳細をご覧ください 。

      4. 地域を選択します。 Cloud Managerが検出したリージョンにボリュームを作成できます。

      5. IPアドレスまたはクラスレスドメイン間ルーティング（CIDR）を指定して、クライアントアクセスを制限します。

    4. Goをクリックします。

    クラウドボリュームの削除

    不要になったクラウドボリュームを削除します。

    手順

    1. 作業環境を開きます。

    2. ボリュームにカーソルを合わせてメニューをクリックします。 削除をクリックします。

    3. ボリュームを削除することを確認します。

    ヘルプを得る

    一般的なサービスに関する質問については、Cloud Managerチャットを使用してください。

    クラウドボリュームに関連するテクニカルサポートの問題については、Cloud Volumes Serviceユーザーインターフェイスの[サポート]タブにある20桁の「930」シリアル番号を使用してください。 Webチケットを開くとき、またはサポートに電話するときに、このサポートIDを使用します。 Cloud Volumes Serviceのユーザーインターフェイスからサポートできるように、Cloud Volumes Serviceのシリアル番号を必ず有効にしてください。 それらのステップはここで説明されます 。

    制限事項

    • Cloud ManagerはSMBまたはデュアルプロトコルボリュームをサポートしていません。

    • クラウドボリュームは、初期サイズと単一のエクスポートポリシーでのみ作成できます。ボリュームの編集は、Cloud Volume Serviceユーザーインターフェイスから行う必要があります。

    • Cloud Managerは、AWSサブスクリプションのCloud Volumesサービスとの間のデータ複製をサポートしていません。

    • Cloud ManagerからAWSサブスクリプションのCloud Volumesサービスを削除することはサポートされていません。 Cloud Managerから地域を発見するための料金はかかりません。
