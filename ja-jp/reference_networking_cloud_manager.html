    Cloud Managerのネットワーク要件 GitHubで編集 ドキュメントの変更をリクエストする

    11/05/2019 寄稿者

    Cloud ManagerがAWS、Microsoft Azure、またはGoogle Cloud PlatformにCloud Volumes ONTAPシステムを展開できるようにネットワークを設定します。最も重要なステップは、さまざまなエンドポイントへのアウトバウンドインターネットアクセスを確保することです。

    ネットワークがインターネットへのすべての通信にプロキシサーバーを使用している場合、Cloud Managerはセットアップ中にプロキシを指定するよう求めます。 [設定]ページからプロキシサーバーを指定することもできます。 プロキシサーバーを使用するためのCloud Managerの構成を参照してください。

    ターゲットネットワークへの接続

    Cloud Managerには、Cloud Volumes ONTAPをデプロイするVPCおよびVNetへのネットワーク接続が必要です。

    たとえば、企業ネットワークにCloud Managerをインストールする場合、Cloud Volumes ONTAPを起動するVPCまたはVNetへのVPN接続を設定する必要があります。

    アウトバウンドインターネットアクセス

    Cloud Managerでは、Cloud Volumes ONTAPを展開および管理するためにアウトバウンドインターネットアクセスが必要です。 WebブラウザーからCloud Managerにアクセスする場合、およびLinuxホストでCloud Managerインストーラーを実行する場合は、アウトバウンドインターネットアクセスも必要です。

    次のセクションでは、特定のエンドポイントを特定します。

    AWSでCloud Volumes ONTAPを管理するエンドポイント

    Cloud Managerは、AWSでCloud Volumes ONTAPをデプロイおよび管理するときに、次のエンドポイントに接続するためにアウトバウンドインターネットアクセスを必要とします。

    エンドポイント目的

    AWSサービス（amazonaws.com）：

    • CloudFormation

    • Elastic Compute Cloud（EC2）

    • キー管理サービス（KMS）

    • セキュリティトークンサービス（STS）

    • シンプルストレージサービス（S3）

    正確なエンドポイントは、Cloud Volumes ONTAPを展開する地域によって異なります。 詳細については、AWSのドキュメントを参照してください。

    Cloud ManagerがCloud Volumes ONTAPをAWSにデプロイおよび管理できるようにします。

    https://api.services.cloud.netapp.com:443

    NetApp Cloud CentralへのAPIリクエスト。

    https://cloud.support.netapp.com.s3.us-west-1.amazonaws.com

    ソフトウェアイメージ、マニフェスト、およびテンプレートへのアクセスを提供します。

    https://cognito-idp.us-east-1.amazonaws.com
    https://cognito-identity.us-east-1.amazonaws.com

    Cloud Managerがマニフェスト、テンプレート、Cloud Volumes ONTAPアップグレードイメージにアクセスしてダウンロードできるようにします。

    https://kinesis.us-east-1.amazonaws.com

    NetAppが監査レコードからデータをストリーミングできるようにします。

    https://cloudmanager.cloud.netapp.com

    Cloud Centralアカウントを含むCloud Managerサービスとの通信。

    https://netapp-cloud-account.auth0.com

    集中ユーザー認証のためのNetApp Cloud Centralとの通信。

    https://w86yt021u5.execute-api.us-east-1.amazonaws.com/production/whitelist

    Cloud Backup Serviceの許可されたユーザーのリストにAWSアカウントIDを追加するために使用されます。

    https://support.netapp.com/aods/asupmessage
    https://support.netapp.com/asupprod/post/1.0/postAsup

    NetApp AutoSupportとの通信。

    https://support.netapp.com/svcgw
    https://support.netapp.com/ServiceGW/entitlement

    システムライセンスおよびサポート登録のためのNetAppとの通信。

    https://ipa-signer.cloudmanager.netapp.com

    Cloud Managerがライセンスを生成できるようにします（たとえば、Cloud Volumes ONTAPのFlexCacheライセンス）

    https://packages.cloud.google.com/yum
    https://github.com/NetApp/trident/releases/download/

    Cloud Volumes ONTAPシステムをKubernetesクラスターに接続するために必要です。エンドポイントにより、NetApp Tridentのインストールが可能になります。

    さまざまなサードパーティの場所、たとえば：

    • https://repo1.maven.org/maven2

    • https://oss.sonatype.org/content/repositories

    • https://repo.typesafe.org

    サードパーティの場所は変更される場合があります。

    アップグレード中に、Cloud Managerはサードパーティの依存関係の最新パッケージをダウンロードします。

    Webブラウザーからアクセスされるエンドポイント

    ユーザーはWebブラウザーからCloud Managerにアクセスする必要があります。 Webブラウザーを実行しているマシンには、次のエンドポイントへの接続が必要です。

    エンドポイント目的

    Cloud Managerホスト

    Cloud Managerコンソールをロードするには、WebブラウザーからホストのIPアドレスを入力する必要があります。

    クラウドプロバイダーへの接続に応じて、ホストに割り当てられたプライベートIPまたはパブリックIPを使用できます。

    • VPNがあり、仮想ネットワークへの直接接続アクセスがある場合、プライベートIPが機能します

    • パブリックIPは、あらゆるネットワークシナリオで機能します

    いずれの場合でも、セキュリティグループルールが許可されたIPまたはサブネットからのアクセスのみを許可するようにすることにより、ネットワークアクセスを保護する必要があります。

    https://auth0.com
    https://cdn.auth0.com
    https://netapp-cloud-account.auth0.com
    https://services.cloud.netapp.com

    Webブラウザーはこれらのエンドポイントに接続して、NetApp Cloud Centralを介した集中ユーザー認証を行います。

    https://widget.intercom.io

    NetAppクラウドの専門家と話すことができる製品内チャットの場合。

    Cloud Managerインストーラーは、インストールプロセス中に次のURLにアクセスする必要があります。

    • http://dev.mysql.com/get/mysql-community-release-el7-5.noarch.rpm

    • https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm

    • https://s3.amazonaws.com/aws-cli/awscli-bundle.zip
