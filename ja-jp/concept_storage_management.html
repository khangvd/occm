    ストレージ管理 GitHubで編集 ドキュメントの変更をリクエストする

    06/06/2019 貢献者

    Cloud Managerは、Cloud Volumes ONTAPストレージの簡素化された高度な管理を提供します。

    すべてのディスクとアグリゲートを作成してCloud Managerから直接削除する必要があります。他の管理ツールからこれらの操作を実行しないでください。これを行うと、システムの安定性に影響を与え、将来的にディスクを追加する能力を妨げ、冗長なクラウドプロバイダー料金を発生させる可能性があります。

    ストレージプロビジョニング

    Cloud Managerは、ディスクを購入してアグリゲートを管理することによって、Cloud Volumes ONTAPのストレージプロビジョニングを容易にします。単にボリュームを作成するだけです。必要に応じて、高度な割り当てオプションを使用して集約を自分でプロビジョニングすることができます。

    簡易プロビジョニング

    アグリゲートはクラウドストレージをボリュームに提供します。 Cloud Managerは、インスタンスを起動したとき、および追加のボリュームをプロビジョニングしたときに、アグリゲートを作成します。

    ボリュームを作成すると、Cloud Managerは3つのうちの1つを行います。

    • 十分な空き容量がある既存のアグリゲートにボリュームを配置します。

    • そのアグリゲート用にさらにディスクを購入することによって、ボリュームを既存のアグリゲートに配置します。

    • 新しいアグリゲート用のディスクを購入し、そのアグリゲートにボリュームを配置します。

    Cloud Managerは、アグリゲートの最大サイズ、シンプロビジョニングが有効になっているかどうか、アグリゲートの空き容量のしきい値など、いくつかの要素を考慮して新しいボリュームの配置場所を決定します。

    アカウント管理者は、 設定ページから空き容量のしきい値を変更できます。

    AWSのアグリゲートのディスクサイズ選択

    Cloud ManagerがAWSのCloud Volumes ONTAP用に新しいアグリゲートを作成すると、システム内のアグリゲート数が増加するにつれて、アグリゲート内のディスクサイズが徐々に増加します。 Cloud Managerは、AWSが許可するデータディスクの最大数に達する前にシステムの最大容量を利用できるようにするためにこれを行います。

    たとえば、Cloud Managerは、Cloud Volumes ONTAP PremiumまたはBYOLシステムのアグリゲートに次のディスクサイズを選択できます。

    高度な割り当てオプションを使用して、自分でディスクサイズを選択できます。

    高度な割り当て

    Cloud Managerにアグリゲートを管理させるのではなく、自分で行うことができます。 [ 詳細割り当て]ページから、特定の数のディスクを含む新しいアグリゲートを作成したり、既存のアグリゲートにディスクを追加したり、特定のアグリゲートにボリュームを作成したりできます。

    容量管理

    Cloud Manager管理者は、Cloud Managerがストレージ容量の決定をユーザーに通知するかどうか、またはCloud Managerが自動的に容量要件を管理するかどうかを選択できます。これらのモードがどのように機能するかを理解するのに役立つかもしれません。

    自動容量管理

    Cloud Manager管理者が容量管理モードを自動に設定した場合、より多くの容量が必要になるとCloud Managerは自動的にCloud Volumes ONTAPインスタンスの新しいディスクを購入し、未使用のディスクのコレクションを削除します。ディスク。

    次の例は、このモードの仕組みを示しています。

    • 5台以下のEBSディスクを含むアグリゲートが容量のしきい値に達すると、Cloud Managerは自動的にそのアグリゲート用の新しいディスクを購入するので、ボリュームは拡大し続けることができます。

    • 12台のAzureディスクを含むアグリゲートが容量のしきい値に達すると、Cloud Managerはそのアグリゲートから利用可能な容量を持つアグリゲートまたは新しいアグリゲートにボリュームを自動的に移動します。

      Cloud Managerがそのボリューム用の新しいアグリゲートを作成する場合、そのボリュームのサイズに対応するディスクサイズを選択します。

      元のアグリゲートで空き容量が利用できるようになったことに注意してください。既存のボリュームまたは新しいボリュームがそのスペースを使用できます。このシナリオでは、スペースをAWSまたはAzureに戻すことはできません。

    • アグリゲートに12時間を超えてボリュームが含まれていない場合、Cloud Managerはそれを削除します。

    手動容量管理

    Cloud Manager管理者が容量管理モードを手動に設定した場合、容量を決定する必要があるときにCloud ManagerはAction Requiredメッセージを表示します。自動モードで説明したのと同じ例が手動モードにも当てはまりますが、アクションを受け入れるのはあなた次第です。
