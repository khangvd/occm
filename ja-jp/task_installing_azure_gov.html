    米国のAzureリージョンにCloud Managerをデプロイする GitHubで編集 ドキュメントの変更をリクエストする

    10/07/2019 寄稿者

    Cloud Governmentを米国政府地域で稼働させるには、まずAzure Government MarketplaceからCloud Managerをデプロイします。次に、Cloud ManagerがCloud Volumes ONTAPシステムをデプロイおよび管理するために必要な権限を提供します。

    サポートされているAzure US Governmentリージョンのリストについては、 Cloud Volumes Global Regionsをご覧ください。

    Azure US Government MarketplaceからCloud Managerをデプロイする

    Cloud Managerは、Azure US Government Marketplaceでイメージとして利用できます。

    手順

    1. サブスクリプションでAzure Government Marketplaceが有効になっていることを確認します。

      1. エンタープライズ管理者としてポータルにログインします。

      2. [管理]に移動します。

      3. [ 登録の詳細 ]で、 Azure Marketplaceの横にある鉛筆アイコンをクリックします。

      4. [ 有効]を選択します。

      5. 保存をクリックします 。

        Microsoft Azureドキュメント：Azure Government Marketplace

    2. Azure US GovernmentポータルでOnCommand Cloud Managerを検索します。

    3. [ 作成]をクリックし、手順に従って仮想マシンを構成します。

      仮想マシンを構成する際には、次のことに注意してください。

      • Cloud Managerは、HDDまたはSSDディスクで最適に実行できます。

      • 推奨される仮想マシンサイズのいずれかを選択する必要があります：A2、D2 v2、またはD2 v3（可用性に基づいて）。

      • ネットワークセキュリティグループの場合は、[ 詳細設定]を選択するのが最適です。

        詳細オプションは、Cloud Managerに必要なインバウンドルールを含む新しいセキュリティグループを作成します。 [基本]を選択した場合、必要なルールのリストについては、 セキュリティグループルールを参照してください。

    4. 概要ページで選択内容を確認し、[ 作成 ]をクリックして展開を開始します 。

      Azureは、指定された設定で仮想マシンをデプロイします。仮想マシンとCloud Managerソフトウェアは、約5分で実行されるはずです。

    5. Cloud Manager仮想マシンに接続しているホストからWebブラウザーを開き、次のURLを入力します。

      http：// ipaddress ：80

    6. ログインした後、Cloud Managerをセットアップします。

      1. このCloud Managerシステムに関連付けるCloud Centralアカウントを指定します。

        Cloud Centralアカウントについて学習します 。

      2. システムの名前を入力します。

    結果

    これで、Cloud Managerがインストールおよびセットアップされました。ユーザーがAzureにCloud Volumes ONTAPを展開する前に、Azureのアクセス許可を付与する必要があります。

    管理されたIDを使用してCloud ManagerにAzureのアクセス許可を付与する

    許可を提供する最も簡単な方法は、Cloud Manager仮想マシンで管理対象IDを有効にしてから、必要な許可を仮想マシンに割り当てることです。必要に応じて、別の方法は、サービスプリンシパルを使用してAzureのアクセス許可を付与することです。

    手順

    1. Cloud Manager仮想マシンで管理IDを有効にします。

      1. Cloud Manager仮想マシンに移動し、 「ID」を選択します。

      2. [ システムの割り当て ]で、[ オン ]をクリックし、[ 保存 ]をクリックします 。

    2. Cloud Managerポリシーを使用してカスタムロールを作成します。

      1. Cloud Manager Azureポリシーをダウンロードします。

      2. AzureサブスクリプションIDを割り当て可能なスコープに追加して、JSONファイルを変更します。

        ユーザーがCloud Volumes ONTAPシステムを作成する各AzureサブスクリプションのIDを追加する必要があります。

        例

        「AssignableScopes」：[
        「/ subscriptions / d333af45-0d07-4154-943d-c25fbzzzzzzz」、
        「/ subscriptions / 54b91999-b3e6-4599-908e-416e0zzzzzzz」、
        「/ subscriptions / 398e471c-3b42-4ae7-9b59-ce5bbzzzzzzz」

      3. JSONファイルを使用して、Azureでカスタムロールを作成します。

        次の例は、Azure CLI 2.0を使用してカスタムロールを作成する方法を示しています。

        azロール定義create --role-definition C：\ Policy_for_cloud_Manager_Azure_3.7.4.json

        これで、Cloud Manager仮想マシンに割り当てることができるOnCommand Cloud Manager Operatorというカスタムロールが必要になります。

    3. 1つ以上のサブスクリプションのロールをCloud Manager仮想マシンに割り当てます。

      1. サブスクリプションサービスを開き、Cloud Volumes ONTAPシステムを展開するサブスクリプションを選択します。

      2. [ アクセス制御（IAM）]をクリックします。

      3. [ 追加 ] をクリックし、[ 役割の割り当てを追加 ] をクリックして、アクセス許可を追加します。

        • OnCommand Cloud Manager Operatorロールを選択します。

          OnCommand Cloud Manager Operatorは、 Cloud Managerポリシーで提供されるデフォルト名です。ロールに別の名前を選択した場合は、代わりにその名前を選択します。

        • 仮想マシンへのアクセスを割り当てます。

        • Cloud Manager仮想マシンが作成されたサブスクリプションを選択します。

        • 仮想マシンの名前を入力して選択します。

        • 保存をクリックします 。

      4. 追加のサブスクリプションからCloud Volumes ONTAPを展開する場合は、そのサブスクリプションに切り替えてから、これらの手順を繰り返します。

    結果

    Cloud Managerには、AzureでCloud Volumes ONTAPをデプロイおよび管理するために必要な権限があります。
