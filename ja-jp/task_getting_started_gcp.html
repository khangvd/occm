    Google Cloud PlatformでCloud Volumes ONTAPを開始する GitHubで編集 ドキュメントの変更をリクエストする

    11/05/2019 寄稿者

    GCPをセットアップしてからNetApp Cloud CentralからCloud Managerソフトウェアを展開することにより、Cloud Volumes ONTAPを始めましょう。

    始める前に、次のことに注意してください。

    • Cloud Volumes ONTAPをGCPにデプロイするには、Cloud ManagerをGoogle Cloud Platformにインストールする必要があります。

    • GCPでCloud Managerを展開する唯一の方法は、NetApp Cloud Centralからです。

    ネットワークをセットアップする

    Cloud ManagerとCloud Volumes ONTAPが複数のエンドポイントに接続できるように、ターゲットVPCからのアウトバウンドインターネットアクセスを有効にします。

    Cloud Managerは、アウトバウンドインターネットアクセスなしではCloud Volumes ONTAPをデプロイできないため、この手順は重要です。アウトバウンド接続を制限する必要がある場合は、 Cloud ManagerおよびCloud Volumes ONTAPのエンドポイントのリストを参照してください。

    データ階層化のためにGCPをセットアップする

    Cloud Volumes ONTAPから低コストのオブジェクトストレージ（Google Cloud Storageバケット）にコールドデータを階層化する場合は、いくつかの手順が必要です。

    1. プライベートGoogleアクセス用にCloud Volumes ONTAPサブネットを構成します 。

    2. 定義済みのStorage Adminロールを持つサービスアカウントを作成します 。

    3. GCPストレージ設定に移動し、サービスアカウントのアクセスキーを作成します。

      • プロジェクトを選択し、 相互運用性をクリックします。まだ行っていない場合は、[ 相互運用性アクセスを有効にする]をクリックします 。

      • [ サービスアカウントのアクセスキー ]で、[ サービスアカウントのキーの 作成 ]をクリックし、 作成したサービスアカウントを選択して、[ キーの作成 ]をクリックします 。

        後でCloud Managerにキーを追加して、キーを使用してデータ階層化のためのCloud Storageバケットを管理できるようにする必要があります。

    Google Cloud APIを有効にする

    プロジェクトで次のGoogle Cloud APIを有効にします 。これらのAPIは、Cloud ManagerとCloud Volumes ONTAPをデプロイするために必要です。

    • Cloud Deployment Manager V2 API

    • Cloud Resource Manager API

    • Compute Engine API

    • Stackdriver Logging API

    GCPのアクセス許可とプロジェクトを設定する

    2セットの許可が設定されていることを確認してください。

    1. NetApp Cloud CentralからCloud ManagerをデプロイするGCPユーザーが、GCPのCloud Centralポリシーで権限を持っていることを確認してください。

      YAMLファイルを使用してカスタムロールを作成し、それをユーザーにアタッチできます。ロールを作成するには、gcloudコマンドラインを使用する必要があります。

    2. プロジェクトでCloud Volumes ONTAPシステムを作成および管理するためにCloud Managerが必要とする権限を持つサービスアカウントを設定します。

      手順6で、このサービスアカウントをCloud Manager VMに関連付けます。このサービスアカウントは、データ階層化に使用されるサービスアカウントとは異なります。

      • GCPのCloud Managerポリシーで定義された権限を含むGCPでロールを作成します 。繰り返しますが、gcloudコマンドラインを使用する必要があります。

        このYAMLファイルに含まれる許可は、ステップ4aの許可とは異なります。

      • GCPサービスアカウントを作成し、作成したカスタムロールを適用します 。

      • Cloud Volumes ONTAPを他のプロジェクトにデプロイする場合は、そのプロジェクトにCloud Managerロールを持つサービスアカウントを追加してアクセスを許可します 。プロジェクトごとにこの手順を繰り返す必要があります。

    GCP Marketplaceから購読する

    GCP MarketplaceからCloud Volumes ONTAPにサブスクライブして 、無料試用期間終了後にサービスが中断しないようにします。作成したCloud Volumes ONTAP PAYGOシステムごとに、このサブスクリプションから請求されます。

    NetApp Cloud CentralからCloud Managerを起動します

    Cloud Volumes ONTAPを展開および管理するには、Cloud Managerソフトウェアが必要です。 Cloud CentralからGCPでCloud Managerインスタンスを起動するのに数分かかります。 Cloud Volumes ONTAPをデプロイするには、Cloud ManagerをCloud CentralからGCPにインストールする必要があります。

    GCPをクラウドプロバイダーとして選択すると、Googleからアカウントにログインして権限を付与するように求められます。 [許可]をクリックすると、Cloud Managerのデプロイに必要なコンピューティングAPIへのアクセスが許可されます。

    Cloud Managerを使用してCloud Volumes ONTAPを起動する

    Cloud Managerの準備ができたら、[作成]をクリックし、展開するシステムのタイプを選択して、ウィザードの手順を完了します。 25分後、最初のCloud Volumes ONTAPシステムが稼働するはずです。

    これらの手順については、次のビデオをご覧ください。
