    Cloud Volumes ONTAPシステムの変更 GitHubで編集 ドキュメントの変更をリクエストする

    09/17/2019 寄稿者

    ストレージのニーズの変化に応じて、Cloud Volumes ONTAPインスタンスの構成を変更する必要がある場合があります。たとえば、従量課金制の設定を変更したり、インスタンスまたはVMの種類を変更したり、代替サブスクリプションに移動したりできます。

    Cloud Volumes ONTAP BYOLシステムへのライセンスファイルのインストール

    Cloud ManagerがNetAppからBYOLライセンスファイルを取得できない場合、自分でファイルを取得し、Cloud Managersにファイルを手動でアップロードして、Cloud Volumes ONTAPシステムにライセンスをインストールできます。

    手順

    1. NetApp License File Generatorに移動し、 NetAppサポートサイトの資格情報を使用してログインします。

    2. パスワードを入力し、製品を選択し、シリアル番号を入力し、プライバシーポリシーを読んで同意したことを確認して、[ 送信 ]をクリックします。

      例

    3. serialnumber.NLF JSONファイルを電子メールで受信するか、直接ダウンロードするかを選択します。

    4. Cloud Managerで、Cloud Volumes ONTAP BYOL作業環境を開きます。

    5. メニューアイコンをクリックし、[ ライセンス ]をクリックします。

    6. [ ライセンスファイルのアップロード]をクリックします 。

    7. [ アップロード]をクリックして、ファイルを選択します。

    結果

    Cloud Managerは、Cloud Volumes ONTAPシステムに新しいライセンスファイルをインストールします。

    Cloud Volumes ONTAPのインスタンスまたはマシンタイプの変更

    AWS、Azure、またはGCPでCloud Volumes ONTAPを起動するときに、いくつかのインスタンスまたはマシンタイプから選択できます。インスタンスまたはマシンのタイプは、必要に応じて小さすぎるまたは大きすぎると判断した場合、いつでも変更できます。

    このタスクについて

    • Cloud Volumes ONTAP HAペアで自動ギブバックを有効にする必要があります（これはデフォルト設定です）。そうでない場合、操作は失敗します。

      ONTAP 9ドキュメント：自動ギブバックを設定するためのコマンド

    • 操作によりCloud Volumes ONTAPが再起動します。

      単一ノードシステムの場合、I / Oは中断されます。

      HAペアの場合、変更は中断されません。 HAペアは引き続きデータを提供します。

    • インスタンスまたはマシンのタイプを変更すると、クラウドプロバイダーのサービス料金に影響します。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[AWSのライセンスまたはインスタンスの変更 ]、[AzureのライセンスまたはVMの変更 ]、 または [GCPのライセンスまたはマシンの変更 ]をクリックします。

    2. 従量課金制を使用している場合は、オプションで別のライセンスを選択できます。

    3. インスタンスまたはマシンのタイプを選択し、チェックボックスを選択して変更の影響を理解していることを確認し、[ OK ]をクリックします。

    結果

    Cloud Volumes ONTAPが新しい構成で再起動します。

    従量課金制の設定を変更する

    従量課金制のCloud Volumes ONTAPシステムを起動した後、ライセンスを変更することで、いつでもExplore、Standard、Premiumの構成を変更できます。ライセンスを変更すると、rawの容量制限が増減し、さまざまなAWSインスタンスタイプまたはAzure仮想マシンタイプから選択できます。

    GCPでは、従量課金制の構成ごとに単一のマシンタイプを使用できます。異なるマシンタイプを選択することはできません。
    このタスクについて

    従量課金ライセンス間の変更については、次のことに注意してください。

    • 操作によりCloud Volumes ONTAPが再起動します。

      単一ノードシステムの場合、I / Oは中断されます。

      HAペアの場合、変更は中断されません。 HAペアは引き続きデータを提供します。

    • インスタンスまたはマシンのタイプを変更すると、クラウドプロバイダーのサービス料金に影響します。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[AWSのライセンスまたはインスタンスの変更 ]、[AzureのライセンスまたはVMの変更 ]、 または [GCPのライセンスまたはマシンの変更 ]をクリックします。

    2. ライセンスタイプとインスタンスタイプまたはマシンタイプを選択し、チェックボックスを選択して変更の影響を理解していることを確認し、[ OK ]をクリックします。

    結果

    Cloud Volumes ONTAPは、新しいライセンス、インスタンスタイプ、マシンタイプ、またはその両方で再起動します。

    別のCloud Volumes ONTAP構成への移行

    Pay-as-you-goサブスクリプションとBYOLサブスクリプション間、または単一のCloud Volumes ONTAPシステムとHAペア間を移動する場合、新しいシステムをデプロイしてから、既存のシステムから新しいシステムにデータを複製できます。 。

    手順

    1. 新しいCloud Volumes ONTAP作業環境を作成します。

      AWSでCloud Volumes ONTAPを起動する
      AzureでCloud Volumes ONTAPを起動する
      GCPでCloud Volumes ONTAPを起動する

    2. 複製する必要があるボリュームごとに、システム間で1回限りのデータ複製をセットアップします。

    3. 元の作業環境を削除して、不要になったCloud Volumes ONTAPシステムを終了します 。

    AWS Marketplaceサブスクリプションを変更する

    請求先のAWSアカウントを変更する場合は、Cloud Volumes ONTAPシステムのAWS Marketplaceサブスクリプションを変更します。

    手順

    1. AWS MarketplaceのCloud Managerオファリングから新しいサブスクリプションをまだ追加していない場合は追加します 。

    2. Cloud Managerの作業環境から、メニューアイコンをクリックしてから、 Marketplace Subscriptionをクリックします。

    3. ドロップダウンリストからサブスクリプションを選択します。

    4. 保存をクリックします 。

    ストレージ仮想マシン名の変更

    Cloud Managerは、Cloud Volumes ONTAPのストレージ仮想マシン（SVM）に自動的に名前を付けます。厳密な命名基準がある場合は、SVMの名前を変更できます。たとえば、ONTAPクラスタのSVMの命名方法と一致させることができます。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[ 情報 ]をクリックします。

    2. SVM名の右側にある編集アイコンをクリックします。

    3. [SVM名の変更]ダイアログボックスで、SVM名を変更し、[ 保存 ]をクリックします 。

    Cloud Volumes ONTAPのパスワードの変更

    Cloud Volumes ONTAPにはクラスター管理者アカウントが含まれています。必要に応じて、Cloud Managerからこのアカウントのパスワードを変更できます。

    System ManagerまたはCLIを使用して管理者アカウントのパスワードを変更しないでください。パスワードはCloud Managerに反映されません。その結果、Cloud Managerはインスタンスを適切に監視できません。
    手順

    1. 作業環境から、メニューアイコンをクリックし、[ 詳細設定 ] > [ パスワードの設定 ]をクリックします 。

    2. 新しいパスワードを2回入力し、[ 保存 ]をクリックします 。

      新しいパスワードは、最後に使用した6つのパスワードのいずれかと異なる必要があります。

    c4.4xlargeおよびc4.8xlargeインスタンスのネットワークMTUの変更

    デフォルトでは、AWSでc4.4xlargeインスタンスまたはc4.8xlargeインスタンスを選択すると、Cloud Volumes ONTAPは9,000 MTU（ジャンボフレームとも呼ばれます）を使用するように構成されます。ネットワーク構成により適している場合は、ネットワークMTUを1,500バイトに変更できます。

    このタスクについて

    9,000バイトのネットワーク最大伝送ユニット（MTU）は、特定の構成で可能な最大の最大ネットワークスループットを提供できます。

    同じVPC内のクライアントがCloud Volumes ONTAPシステムと通信し、それらのクライアントの一部またはすべてが9,000 MTUもサポートする場合、9,000 MTUが適切な選択です。トラフィックがVPCを離れると、パケットの断片化が発生し、パフォーマンスが低下します。

    1,500バイトのネットワークMTUは、VPCの外部のクライアントまたはシステムがCloud Volumes ONTAPシステムと通信する場合に適しています。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[ 詳細設定 ] > [ネットワーク使用率 ]をクリックします。

    2. [ 標準]または[ ジャンボフレーム]を選択します。

    3. 変更をクリックします。

    複数のAWS AZのHAペアに関連付けられたルートテーブルの変更

    HAペアのフローティングIPアドレスへのルートを含むAWSルートテーブルを変更できます。新しいNFSまたはCIFSクライアントがAWSのHAペアにアクセスする必要がある場合、これを行うことができます。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[ 情報 ]をクリックします。

    2. [ ルートテーブル]をクリックします。

    3. 選択したルートテーブルのリストを変更し、[ 保存 ]をクリックします 。

    結果

    Cloud ManagerはAWSリクエストを送信して、ルートテーブルを変更します。
