    非アクティブなデータを低コストのオブジェクトストレージに階層化する GitHubで編集 ドキュメントの変更をリクエストする

    01/08/2020 寄稿者

    ホットデータ用のSSDまたはHDDパフォーマンス層を非アクティブデータ用のオブジェクトストレージ容量層と組み合わせることにより、ストレージコストを削減できます。高レベルの概要については、 データ階層化の概要を参照してください。

    データ階層化を設定するには、次のことを行うだけです。

    サポートされている構成を選択してください

    ほとんどの構成がサポートされています。最新バージョンを実行しているCloud Volumes ONTAP Standard、Premium、またはBYOLシステムがある場合は、準備完了です。 詳細をご覧ください 。

    Cloud Volumes ONTAPとオブジェクトストレージ間の接続を確保する

    • AWSの場合、S3へのVPCエンドポイントが必要です。 詳細をご覧ください 。

    • Azureの場合、Cloud Managerに必要な権限がある限り、何もする必要はありません。 詳細をご覧ください 。

    • GCPの場合、プライベートGoogleアクセスのサブネットを構成し、サービスアカウントをセットアップする必要があります。 詳細をご覧ください 。

    ボリュームを作成、変更、または複製するときに階層化ポリシーを選択する

    Cloud Managerは、ボリュームを作成、変更、または複製するときに、階層化ポリシーを選択するように求めます。

    • 読み書きボリューム上のデータの階層化

    • データ保護ボリューム上のデータの階層化

    データ階層化に不要なもの

    • データ階層化を有効にするために機能ライセンスをインストールする必要はありません。

    • 容量層（S3バケット、Azure Blobコンテナー、またはGCPバケット）を作成する必要はありません。 Cloud Managerがそれを行います。

    データ階層化をサポートする構成

    特定の構成と機能を使用する場合、データの階層化を有効にできます。

    • データ階層化は、次のバージョン以降のCloud Volumes ONTAP Standard、Premium、およびBYOLでサポートされています。

      • AWSのバージョン9.2

      • シングルノードシステムを使用するAzureのバージョン9.4

      • HAペアを備えたAzureのバージョン9.6

      • GCPのバージョン9.6

        DS3_v2仮想マシンタイプのAzureでは、データ階層化はサポートされていません。

    • AWSでは、パフォーマンス層は汎用SSD、プロビジョニングされたIOPS SSD、スループット最適化HDDのいずれかです。

    • Azureでは、パフォーマンス層は、プレミアムSSD管理ディスク、標準SSD管理ディスク、または標準HDD管理ディスクにすることができます。

    • GCPでは、パフォーマンス層はSSDまたはHDD（標準ディスク）のいずれかです。

    • データ階層化は暗号化技術でサポートされています。

    • ボリュームでシンプロビジョニングを有効にする必要があります。

    コールドデータをAWS S3に階層化するための要件

    Cloud Volumes ONTAPにS3への接続があることを確認します。その接続を提供する最良の方法は、S3サービスへのVPCエンドポイントを作成することです。手順については、「 AWSドキュメント：ゲートウェイエンドポイントの作成」を参照してください。

    VPCエンドポイントを作成するときは、Cloud Volumes ONTAPインスタンスに対応するリージョン、VPC、およびルートテーブルを必ず選択してください。また、セキュリティグループを変更して、S3エンドポイントへのトラフィックを有効にするアウトバウンドHTTPSルールを追加する必要があります。そうしないと、Cloud Volumes ONTAPはS3サービスに接続できません。

    問題が発生した場合は、 AWS Support Knowledge Center：ゲートウェイVPCエンドポイントを使用してS3バケットに接続できないのはなぜですか？を参照してください。 。

    コールドデータをAzure Blobストレージに階層化するための要件

    Cloud Managerに必要なアクセス許可がある限り、パフォーマンス層と容量層の間に接続を設定する必要はありません。 Cloud Managerポリシーに次の権限がある場合、Cloud ManagerはVNetサービスエンドポイントを有効にします。 

     "Microsoft.Network/virtualNetworks/subnets/write", "Microsoft.Network/routeTables/join/action",

    許可は最新のCloud Managerポリシーに含まれています。

    コールドデータをGoogle Cloud Storageバケットに階層化するための要件

    • Cloud Volumes ONTAPが存在するサブネットは、プライベートGoogleアクセス用に構成する必要があります。手順については、 Google Cloudドキュメント：プライベートGoogleアクセスの設定を参照してください 。

    • Cloud Volumes ONTAP 9.7：定義済みのストレージ管理者ロールを持つサービスアカウントを作成する必要があります。 Cloud Volumes ONTAPの作業環境を作成するときに、このサービスアカウントを選択する必要があります。詳細については、 Google Cloud PlatformでCloud Volumes ONTAPを始めようのステップ3をご覧ください。

      Cloud Volumes ONTAP 9.7システムの作成時にデータ階層化を有効にしてサービスアカウントを選択しない場合、システムをオフにし、GCPコンソールからCloud Volumes ONTAPにサービスアカウントを追加する必要があります。

    • Cloud Volumes ONTAP 9.6：サービスアカウントのストレージアクセスキーを入力して、Cloud ManagerにGoogle Cloud Platformアカウントを追加する必要があります。これらのキーにより、Cloud Managerはデータ階層化のためにCloud Storageバケットを設定できます。手順については、 9.6でのデータ階層化のためのGCPアカウントのセットアップと追加を参照してください。

    読み書きボリュームからのデータの階層化

    Cloud Volumes ONTAPは、非アクティブなデータを読み取り/書き込みボリューム上で費用対効果の高いオブジェクトストレージに階層化して、ホットデータ用にパフォーマンス層を解放できます。

    手順

    1. 作業環境で、新しいボリュームを作成するか、既存のボリュームの階層を変更します。

      仕事アクション

      新しいボリュームを作成する

      [ 新しいボリュームの追加]をクリックします。

      既存のボリュームを変更する

      ボリュームを選択し、[ ディスクタイプと階層化ポリシーの変更 ]をクリックします。

    2. [スナップショットのみ]ポリシーまたは[自動]ポリシーを選択します。

      これらのポリシーの説明については、 データ階層化の概要を参照してください。

      例

      Cloud Managerは、データ階層化対応のアグリゲートがまだ存在しない場合、ボリュームの新しいアグリゲートを作成します。

      自分でアグリゲートを作成したい場合は、アグリゲートを作成するときにアグリゲートでデータ階層化を有効にできます。

    データ保護ボリュームからのデータの階層化

    Cloud Volumes ONTAPは、データをデータ保護ボリュームから容量ティアに階層化できます。宛先ボリュームをアクティブにすると、データは読み取られるにつれて徐々にパフォーマンス層に移動します。

    手順

    1. [作業環境]ページで、ソースボリュームを含む作業環境を選択し、ボリュームをレプリケートする作業環境にドラッグします。

    2. 階層化ページに到達するまでプロンプトに従い、オブジェクトストレージへのデータ階層化を有効にします。

      例

      データの複製に関するヘルプについては、クラウドとのデータの複製を参照してください。

    階層化データのストレージクラスの変更

    Cloud Volumes ONTAPを展開した後、30日間アクセスされていない非アクティブなデータのストレージクラスを変更することにより、ストレージコストを削減できます。データにアクセスする場合、アクセスコストが高くなるため、ストレージクラスを変更する前にそれを考慮する必要があります。

    階層化されたデータのストレージクラスはシステム全体であり、ボリュームごとではありません。

    サポートされるストレージクラスについては、 データ階層化の概要を参照してください。

    手順

    1. 作業環境から、メニューアイコンをクリックし、[ ストレージクラス ]または[ Blob Storage Tiering ]をクリックします。

    2. ストレージクラスを選択し、[ 保存 ]をクリックします 。
