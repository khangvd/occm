    GCPでCloud Volumes ONTAPを起動する GitHubで編集 ドキュメントの変更をリクエストする

    10/14/2019 寄稿者

    作業環境を作成することにより、GCPで単一ノードのCloud Volumes ONTAPシステムを起動できます。

    あなたが始める前に

    • 設定を選択し、管理者からGCPネットワーク情報を取得する必要があります。詳細については、 Cloud Volumes ONTAP構成の計画を参照してください。

    • BYOLシステムを展開するには、各ノードに20桁のシリアル番号（ライセンスキー）が必要です。

    手順

    1. [Working Environments]ページで、[ Create Cloud Volumes ONTAP ]をクリックし、プロンプトに従います。

    2. 作業環境の定義 ：[ 続行 ]をクリックします。

    3. Cloud Volumes ONTAPをサブスクライブする：プロンプトが表示されたら、GCP MarketplaceでCloud Volumes ONTAPをサブスクライブします。

      次のビデオは、サブスクリプションプロセスを示しています。

    4. 詳細と資格情報 ：プロジェクトを選択し、クラスター名を指定し、オプションでラベルを追加して、資格情報を指定します。

      次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      Google Cloud Project

      Cloud Volumes ONTAPを配置するプロジェクトを選択します。デフォルトのプロジェクトは、Cloud Managerが存在するプロジェクトです。

      ドロップダウンリストに追加のプロジェクトが表示されない場合は、Cloud Managerサービスアカウントを他のプロジェクトにまだ関連付けていません。 Google Cloudコンソールに移動し、IAMサービスを開き、プロジェクトを選択します。 Cloud Managerロールを持つサービスアカウントをそのプロジェクトに追加します。プロジェクトごとにこの手順を繰り返す必要があります。

      これは、 このページのステップ4bで説明されているように 、Cloud Manager用に設定したサービスアカウントです。

      作業環境名

      Cloud Managerは、作業環境名を使用して、Cloud Volumes ONTAPシステムとGCP VMインスタンスの両方に名前を付けます。また、そのオプションを選択した場合、定義済みのセキュリティグループのプレフィックスとして名前を使用します。

      ラベルを追加

      ラベルは、GCPリソースのメタデータです。 Cloud Managerは、ラベルをCloud Volumes ONTAPシステムおよびシステムに関連付けられたGCPリソースに追加します。

      作業環境を作成するときに、ユーザーインターフェイスから最大4つのラベルを追加し、作成後にさらに追加できます。 APIは、作業環境を作成するときに4つのラベルに制限しないことに注意してください。

      ラベルの詳細については、 Google Cloud Documentation：Labeling Resourcesを参照してください 。

      資格情報

      これらは、Cloud Volumes ONTAPクラスター管理者アカウントの資格情報です。これらの認証情報を使用して、System ManagerまたはそのCLIを介してCloud Volumes ONTAPに接続できます。

    5. 場所と接続性 ：場所を選択し、ファイアウォールポリシーを選択し、チェックボックスを選択して、データ階層化のためのGoogle Cloudストレージへのネットワーク接続を確認します。

      コールドデータをGoogle Cloud Storageバケットに階層化する場合は、Cloud Volumes ONTAPが存在するサブネットをプライベートGoogleアクセス用に構成する必要があります。手順については、 Google Cloudドキュメント：プライベートGoogleアクセスの設定を参照してください 。

    6. ライセンス＆サポートサイトアカウント ：従量課金またはBYOLのどちらを使用するかを指定してから、NetAppサポートサイトアカウントを指定します。

      ライセンスの仕組みを理解するには、参照のライセンスを 。

      ネットアップサポートサイトアカウントは、従量課金制ではオプションですが、BYOLシステムでは必須です。 ネットアップサポートサイトアカウントを追加する方法を学びます 。

    7. 事前構成済みパッケージ ： パッケージの 1つを選択してCloud Volumes ONTAPシステムを迅速に展開するか、[ 独自の構成を作成 ]をクリックします 。

      パッケージのいずれかを選択した場合、ボリュームを指定して、構成を確認および承認するだけです。

    8. ライセンス ：必要に応じてCloud Volumes ONTAPバージョンを変更し、ライセンスを選択して、仮想マシンのタイプを選択します。

      システムの起動後にニーズが変わった場合は、後でライセンスまたは仮想マシンのタイプを変更できます。

      選択したバージョンで新しいリリース候補、一般提供、またはパッチリリースが利用可能な場合、Cloud Managerは作業環境の作成時にシステムをそのバージョンに更新します。たとえば、Cloud Volumes ONTAP 9.5 RC1を選択し、9.5 GAが利用可能な場合、更新が発生します。更新は、リリース間では発生しません（9.4から9.5など）。

    9. 基礎となるストレージリソース ：初期アグリゲートの設定を選択します。ディスクタイプ、各ディスクのサイズ、データ階層化を有効にするかどうか。

      ディスクタイプは初期ボリューム用です。後続のボリュームに別のディスクタイプを選択できます。

      ディスクサイズは、初期アグリゲートのすべてのディスクと、シンプルプロビジョニングオプションを使用するときにCloud Managerが作成する追加アグリゲート用です。拡張割り当てオプションを使用して、異なるディスクサイズを使用するアグリゲートを作成できます。

      ディスクのタイプとサイズの選択については、GCPでのシステムのサイズ設定をご覧ください。

    10. WORM ：必要に応じて、1回書き込み、多数読み取り（WORM）ストレージをアクティブにします。

      WORMストレージの詳細をご覧ください 。

    11. ボリュームの作成 ：新しいボリュームの詳細を入力するか、[ スキップ ]をクリックします。

      iSCSIを使用する場合は、この手順をスキップする必要があります。 Cloud Managerでは、NFSおよびCIFSのみのボリュームを作成できます。

      このページのフィールドの一部は、一目瞭然です。次の表に、ガイダンスが必要なフィールドを示します。

      フィールド説明

      サイズ

      入力できる最大サイズは、シンプロビジョニングを有効にするかどうかによって大きく異なります。これにより、現在使用可能な物理ストレージよりも大きいボリュームを作成できます。

      アクセス制御（NFSのみ）

      エクスポートポリシーは、ボリュームにアクセスできるサブネット内のクライアントを定義します。デフォルトでは、Cloud Managerはサブネット内のすべてのインスタンスへのアクセスを提供する値を入力します。

      権限とユーザー/グループ（CIFSのみ）

      これらのフィールドを使用すると、ユーザーおよびグループの共有へのアクセスレベルを制御できます（アクセス制御リストまたはACLとも呼ばれます）。ローカルまたはドメインのWindowsユーザーまたはグループ、またはUNIXユーザーまたはグループを指定できます。ドメインのWindowsユーザー名を指定する場合は、domain \ usernameの形式を使用してユーザーのドメインを含める必要があります。

      スナップショットポリシー

      Snapshotコピーポリシーは、自動的に作成されるNetApp Snapshotコピーの頻度と数を指定します。 NetApp Snapshotコピーは、パフォーマンスへの影響がなく、必要なストレージが最小限のポイントインタイムファイルシステムイメージです。デフォルトのポリシーを選択することも、何も選択しないこともできます。一時データにはnoneを選択できます。たとえば、Microsoft SQL Serverのtempdbです。

      次の画像は、CIFSプロトコル用に記入されたボリュームページを示しています。

    12. CIFSセットアップ ：CIFSプロトコルを選択した場合は、CIFSサーバーをセットアップします。

      フィールド説明

      DNSプライマリおよびセカンダリIPアドレス

      CIFSサーバーの名前解決を提供するDNSサーバーのIPアドレス。
      リストされたDNSサーバーには、CIFSサーバーが参加するドメインのActive Directory LDAPサーバーとドメインコントローラーを見つけるために必要なサービスロケーションレコード（SRV）が含まれている必要があります。

      参加するActive Directoryドメイン

      CIFSサーバーを参加させるActive Directory（AD）ドメインのFQDN。

      ドメインへの参加を許可された資格情報

      ADドメイン内の指定された組織単位（OU）にコンピューターを追加するための十分な特権を持つWindowsアカウントの名前とパスワード。

      CIFSサーバーのNetBIOS名

      ADドメイン内で一意のCIFSサーバー名。

      組織単位

      CIFSサーバーに関連付けるADドメイン内の組織単位。デフォルトはCN = Computersです。

      DNSドメイン

      Cloud Volumes ONTAPストレージ仮想マシン（SVM）のDNSドメイン。ほとんどの場合、ドメインはADドメインと同じです。

      NTPサーバー

      [ Active Directoryドメインを使用]を選択して、Active Directory DNSを使用してNTPサーバーを構成します。別のアドレスを使用してNTPサーバーを構成する必要がある場合は、APIを使用する必要があります。詳細については、 Cloud Manager API開発者ガイドを参照してください。

    13. 使用プロファイル、ディスクタイプ、および階層化ポリシー ：ストレージ効率機能を有効にするかどうかを選択し、必要に応じて階層化ポリシーを変更します。

      詳細については、 「ボリューム使用プロファイルについて」および「 データ階層化の概要 」を参照してください。

    14. データ階層化のためのGoogle Cloud Platformアカウント ： Google Cloud Platformアカウントに相互運用可能なストレージアクセスキーを提供することにより、データ階層化を設定します。 [ スキップ]をクリックして、データ階層化を無効にします。

      これらのキーにより、Cloud Managerはデータ階層化のためにCloud Storageバケットを設定できます。詳細については、 GCPアカウントのセットアップとCloud Managerへの追加を参照してください。

    15. レビューと承認 ：選択内容をレビューして確認します。

      1. 構成に関する詳細を確認します。

      2. [ 詳細]をクリックして、Cloud Managerが購入するサポートとGCPリソースの詳細を確認します。

      3. チェックボックスをオン...私は理解して選択します。

      4. Goをクリックします。

    結果

    Cloud ManagerはCloud Volumes ONTAPシステムを展開します。タイムラインで進行状況を追跡できます。

    Cloud Volumes ONTAPシステムの展開で問題が発生した場合は、エラーメッセージを確認してください。作業環境を選択して、[環境の再作成 ]をクリックすることもできます 。

    さらにヘルプが必要な場合は、 NetApp Cloud Volumes ONTAP Supportにアクセスしてください。

    終わったら

    • CIFS共有をプロビジョニングした場合は、ユーザーまたはグループにファイルとフォルダーへのアクセス許可を付与し、それらのユーザーが共有にアクセスしてファイルを作成できることを確認します。

    • ボリュームにクォータを適用する場合は、System ManagerまたはCLIを使用します。

      クォータを使用すると、ユーザー、グループ、またはqtreeが使用するディスク容量とファイル数を制限または追跡できます。
